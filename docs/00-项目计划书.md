**版本 v1.0 · 2026-07-02**
**产品定位:高考物理总复习 AI 私教(网页版)——有记忆、有教学法、会归因错题、按薄弱点安排复习,目标是替代线下补课。**

## 第一部分 · 直接回答你的六个问题

### Q1:知识点怎么"植入"系统?

**答:知识不"植入"模型,而是做成一批人工审核过的"知识卡片"文件,每次讲课时把相关卡片喂给模型。**

- 物理形态:项目文件夹里约 250 个(完整版约 900 个)结构化文本文件,每个文件是一张卡片,你用记事本就能打开读。系统会自动生成排版好的阅读版供你审核。
- 一张卡片 = 一个知识点的全部教学资产:定义、公式及适用条件、前置知识、常见误区、**超纲边界(考什么/不考什么/边缘情况)**、关联模型、典型题型、讲法要点。
- 层级:5 大板块 → 6 个课标模块 → 约 30 个单元 → **约 200 个知识点(系统的原子单位)** → 细目(易错点/临界条件/二级结论/物理模型/题型)。
- 讲课时,系统检索出当前话题的卡片(≤5 张)注入给 AI,并硬性要求:**卡片没有覆盖的事实性内容,AI 必须说"我需要核实"而不是凭记忆回答。** 模型是嘴,卡片是脑和缰绳。

为什么这样做:模型的物理知识远超高中(大学、竞赛全都会),它的问题不是"不会",是**不知道边界、每次都即兴发挥、错了无法定位修复**。卡片制让每一次讲课都有可查的依据——发现讲错,修一张卡,全系统永久生效。

### Q2:学生发一个文件,能不能"完美提取所有知识点、有逻辑地构建教学"?

**答:能做到"可靠+可纠错",做不到"完美"。任何现有技术都做不到,承诺完美的产品在撒谎。**

- 关键架构:**"识别 + 映射",不是"自由提取"。** 系统里先有你审核过的约 200 个考点的权威清单,AI 的任务只是"分拣员"——看每道题,从清单里挑出它属于哪几个格子。AI 不许发明新知识点。这保证了:不会提取出超纲的东西(清单里没有)、"漏了什么"清晰可见(清单上没点亮的格子)、错误可度量可回归测试(这是个分类问题)。
- 诚实的准确率预期(未经实测的目标值,上线后以自建测试集实测为准):粘贴文字和清晰印刷体最可靠;**手写错题本明显不可靠,必须走确认流程**;复杂图形题、多考点综合题的归因是天然模糊区。
- 兜底机制(这是比准确率更重要的设计):按"多次独立采样是否一致"分三档——高置信自动入库;中置信让学生 30 秒一键确认("我们认为第 12 题考的是法拉第定律,因为题面要求计算感应电动势——对吗?");低置信进你的复核队列,期间只作为板块级粗信号,不污染精确计划。
- "有逻辑地构建教学":逻辑不是 AI 现场想的。每个考点带前置依赖,构成一张你审核过的知识依赖图;拿到薄弱考点集合后,**确定性算法**(不是 AI)沿依赖图回溯"疑似地基问题"→ 先出几道小题探底 → 拓扑排序 → 生成"地基修复→单点核心→综合应用"的计划,每一步都能解释"因为你第 X 题错了/因为它是 Y 的前置"。
- 骨架版刻意收窄:先只支持**粘贴文字 + 清晰照片单题**(覆盖"拍一道错题发过来"的最高频场景),整卷 PDF、手写整本、批改痕迹识别按"每次只引入一个新失败源"的节奏逐层加。

### Q3:如何做到不漏、不超纲?

**答:把"边界"从模型脑子里搬到人工审核过的文件里。不漏靠机器可执行的覆盖率检查,不超纲靠写在卡片上的边界字段 + 三道防线。**

先说三个调研核实过的制度事实(这是整个方案的地基):

1. **高考考试大纲已于 2019 年底废除**,现行命题依据 = 《普通高中物理课程标准》+《中国高考评价体系》。课标最新版本是 **2017 年版 2025 年修订(2025 年 10 月印发)**,全文已可获取,纸质版可购买。⚠️ 网上大量"2025 修订版解读"是 AI 生成的垃圾内容,**只信官方 PDF,不信任何解读文章**。
2. **新高考物理由各省自主命题**(全国每年 20+ 套物理卷),没有全国统一物理卷。考点集合全国统一(同一课标),但题型结构、深度、评分细则因省而异——所以**必须先锁定一个目标省份**。
3. 5 个版本教材(人教/粤教/鲁科/教科/沪科教)全部依同一课标编写、知识范围一致,差异只在章节顺序和例题——知识库以课标为锚即天然版本无关,教材版本只是一张"目录翻译表"(骨架版只做人教版)。

**不漏的三道防线**:①覆盖率矩阵——每张卡必须引用课标"内容要求"条目原文(逐字从官方 PDF 复制,禁止 AI 默写),脚本每天检查每条课标条目是否至少被一张卡覆盖,目标 100%,这是机器可证明的;②人教版教材目录逐节映射,补齐粒度;③**真题反向抽检**——拿完整真题卷逐题问"这题的考法在库里吗",映射不上的就是漏洞。诚实声明:不存在数学意义上"证明不漏"的方法(官方没有考点总名单),能承诺的是课标覆盖率 100% + 抽检真题可映射率 ≥98%,残余风险靠每年更新真题清单收敛。

**不超纲的三道防线**:①每张卡的 boundary 字段(考什么/不考什么及理由/边缘情况)——这是全卡**唯一必须 100% 人审**的字段,因为"考不考"不是物理知识,是考试制度知识,恰是 AI 最容易错的地方;②全局红线清单注入(洛必达、积分运算、高斯定理、角动量、基尔霍夫、非惯性系等,附课标内替代方法;另有灰色区策略:求导求极值、微元法思想、二级结论的"选择题可用、大题先推导"规则);③生成后自检——每次输出先由第二次轻量调用对照卡片边界检查,违规则重写或拦截。**重要修正:学生问到超纲方法时的正确话术是"高考标准答案不采用、过程不严谨拿不到步骤分、有更稳的课标内方法",而不是"写了不得分"(那是会被任何一线教师戳穿的错误说法)。**

诚实声明:三层防线叠加后漏出率会很低,但**不是零**(三层高度相关,检索漏卡时两层同时失效)。最终兜底:学生端"这个我们没学过"一键反馈按钮 + 固定的 50 问"钓鱼测试集"(故意诱导它用高级方法),每次改提示词或换模型都重跑。

### Q4:如何掌握高中物理"全部细微的知识"?

**答:模型早就"会"全部物理;产品要补的是三样它没有的东西——清单(保证全)、边界(保证不超)、细微处的教学经验(保证像名师)。**

细微知识(分数差距的真正来源)在系统里不是卡片的附属品,而是一等公民,三类实体:

- **易错点**:每条带"错误认识→正确认识→诊断信号→触发特征"。触发特征让易错点能被题目特征**主动唤醒**——学生做传送带题走到共速那一步之前,AI 用提问触发:"到达共速那一刻,摩擦力还在吗?"(埋在台阶上,不是开场白)。
- **模型卡**(传送带/板块/绳杆圆周/弹簧连接体/人船……20-30 张):带结构化的临界条件表("v物=v带的瞬间:水平带摩擦力消失;斜面带要比较 tanθ 与 μ")和隐含条件表("'传送带足够长'意味着必能共速")。
- **二级结论卡**:每条带使用政策("选择题可直接用提速;大题必须写推导链")——这对齐官方命题趋势(教育部考试院连续两年评析点名反对"背结论套模板")。

冷启动诚实规则:没有真实数据前,AI 只许说"很多同学在这里会漏判"(定性),**禁止编造"80% 的学生会错"这类数字**;每条易错点强制标注证据来源(教研判断/公开出处/自有统计),等真实答题数据积累后话术自动升级为真实比例——到那时这反而是竞品没有的资产。这套"边界层+易错点层+讲法层"就是产品的护城河:物理知识满世界都是,这三层没有现成的。

### Q5:练习题如何呈现?

**答:公式用 KaTeX 渲染,示意图用"参数化图形库"程序绘制(AI 只填参数不画图),作答分步进行,判分能用代码的绝不用 AI,AI 批改必须先喂人工标准答案。**

- **公式**:题目统一存 Markdown+LaTeX,前端 KaTeX 渲染(可汗学院出品,高中物理复杂度绰绰有余)。学生输入公式不用学 LaTeX:简化约定(`v0^2=2gh`)+ 实时预览 + 物理高频符号快捷条(θ μ ω ρ √ ²…)。
- **示意图(呈现层最大技术风险,已定架构)**:AI 徒手画 SVG,受力图勉强、**电路图和光路图不可接受**(符号不合国标、角度是"大概齐")。方案是预制参数化图形库:AI 只输出场景 JSON(斜面角度 37°、物块受哪几个力),确定性渲染器画图——程序组装的图,角度、比例、符号永远是对的。骨架版做 6 个高频场景(函数图像/水平物块/斜面/传送带/板块/基础电路——电路用固定拓扑模板,不做自由布线);内测阶段可直接贴真题截图兜底。
- **作答**:选择题点选;数值填空"数值+单位分离"(单位换算由程序做,消灭"答案对但单位写法不同被判错");计算大题**分步作答**(受力分析勾选 → 列方程 → 算结果,逐步解锁)——分步既是教学法(逼学生走规范路径),又让批改难度大幅下降。三级阶梯提示(思路→方法→完整解析),看完解析的题记 0 分并自动安排 3 天后变式重练(防"抄解析假掌握")。
- **批改**:选择题/数值填空纯代码判分(占判分量 60-70%,像计算器一样可靠);计算题过程批改必须注入人工审核过的标准答案+分步评分点(这是可汗学院用真金白银验证的结论,裸批是幻觉重灾区);数值对错由代码复核,不信模型口算;错因归类锁死在固定标签集内(骨架版 4 大类约 20 子码:概念/建模/计算/审题)。四道兜底:非常规解法豁免(学生用动量做、标答用能量做→不判错,转人工)、"我认为批错了"按钮、多次采样一致性做置信分级(低置信不说死话)、批改金标集回归测试。**修正:不信 LLM 自报的置信度数字,置信一律来自多次独立批改的一致性。**
- **题目来源(审查发现的死结,已定方案)**:骨架版 = 你精选真题/经典题 150-300 道(内测自用风险低)+ AI 辅助结构化录入 + 全量人审;变式题用"模板级数字重抽样 + 程序验算 + 抽检"服务复练队列;情境换皮类必须全审;AI 原创题骨架版不做。商用前把真题替换为自编/改编题——你写的解析、错因标注、考点体系是自有资产,题目本体不是。
- **难度与出题**:五档难度对齐"基础:中档:难≈3:5:2";会话内用透明规则引擎调档(连对两题升档、连错三次**停止喂题、退回讲课**——这是私教和题海的分界线);错题本按考点聚类 + 时间线两个视图,复练队列按遗忘曲线(1/3/7/15/30 天)出变式题。

### Q6:"我直接用你 Claude 不可以吗?"

**答:你现在用的我(Claude Code)、你聊天用的 claude.ai、产品要用的 Claude API,是同一个大脑装在三个不同的壳里。产品的本质就是"Claude API + 你自己造的壳",壳里的四样东西恰恰是裸 Claude 没有的。**

裸用 Claude 的六个具体翻车现场 → 每个对应一个要造的组件:

| 翻车现场 | 缺什么 | 要造的组件 |
|---|---|---|
| 周一说过"万有引力很差",周三它全忘了 | API 无状态,零跨会话记忆 | 学生档案库(掌握度/错题史/会话摘要),每次对话注入 |
| 讲动量顺手用微积分推导 | 不知道课标边界 | 知识卡片 + 超纲三防线 |
| 讲得很好,但永远不会主动说"今天该补什么" | 被动应答,没有教学主权 | 教学流程状态机 + 计划引擎(产品定环节,模型只执行当前环节) |
| 批改把对判错、算错数 | 幻觉 + 口算不可靠 | 标答注入批改管线 + 计算器工具 + 代码判分 |
| 没法规范地"做 5 道题、错题进错题本" | 聊天框不是练习界面 | 答题卡/拍照/公式渲染/错题本前端 |
| 不能改界面、接题库、发账号、收费 | claude.ai 是别人的产品 | 整个 app 本身 |

一句话:**模型提供知识和表达力;产品提供教学的行为方式和边界。** 特级教师和名校毕业生的差距从来不在知识量,在于知道大纲边界、记得学生底细、有教学章法、会主动安排下一步——这四样是"工作制度"给的,我们用代码和数据把工作制度固化下来。

**成本(按 2026-06 官方价格测算,量级可信)**:讲课/批改用中档模型(输入 $3/百万 token,2026-08-31 前优惠价 $2)、每日归因用高档模型($5/$25)、转写/打标签用轻量模型($1/$5),配合提示词缓存与批量接口五折。一个学生每天学 1 小时,AI 成本约 **¥3-8/小时,每月约 ¥90-250**——是线下一对一(¥200-500/小时,月 ¥6000-15000)的 **1%-2%**。你自测阶段每月几十到一两百元人民币。成本结构上这个产品成立;真正的风险从来不在 API 账单,在教学质量和下面的风险清单。

## 第二部分 · 系统总架构

```
 学生(浏览器)
    │  看讲解 / 答题 / 拍照
    ▼
┌──────────────────── 你的网页应用 ────────────────────┐
│ 前端:聊天讲课 │ 答题卡 │ 拍照上传 │ 错题本/报告 │ KaTeX │
├──────────────────────────────────────────────────────┤
│ 后端(教学主权在这里,不在模型):                       │
│   状态机:诊断→计划→讲课→练习→批改→归因→调计划       │
│   每次调模型前"备课":取知识卡+标答+学生档案摘要拼成教案 │
│   拿到回复后"记录":更新掌握度、错题本、计划            │
├──────────┬──────────────┬────────────────────────────┤
│ 知识库(文件)│ 学生记忆(数据库)│ 模型适配层(可换发动机)      │
│ 课标条目库  │ 档案/掌握度    │ Claude API(现在)           │
│ 知识卡片    │ 错题史/会话史  │ 国产模型(合规需要时切换)     │
│ 题库+标答   │ 学习计划      │                            │
└──────────┴──────────────┴────────────────────────────┘
```

技术选型(骨架版):Next.js + TypeScript 单仓库;SQLite(学生数据);知识库 = git 仓库内的 JSON/YAML/Markdown 文件(你能直接读、能一键回滚);KaTeX;Anthropic 官方 SDK。全部跑在你本机,唯一外部依赖是模型 API 调用。两条从第一天就执行的纪律:①任何业务代码不得直接依赖特定模型厂商(全部走适配层,换模型=改一张路由表);②数据访问全部走统一数据层(换数据库=改配置)。**除这两个预留外,骨架阶段不为上线提前做任何事——骨架阶段的敌人是复杂度。**

**模型分工**:讲课对话与批改 → 中档模型(质量够、量大省钱);每日错题归因与计划调整 → 高档模型(每天 1 次、推理深,私教含金量所在);拍照转写、打标签、课后总结 → 轻量模型;夜间批量任务走批量接口(五折)。自测起步先全用中档跑通,再分层优化。

| 6 | 教学主权在后端状态机;模型只执行当前环节的单一任务;计划排序用确定性算法不用 AI | 可解释、可测试、不抽风 |
| 7 | 能用代码判的绝不用 AI 判;AI 批改必须注入人工标准答案;数值一律代码复核;表达式等价用现成 CAS(SymPy/mathjs) | 可靠性基本盘 |
| 8 | 置信度一律来自多次独立采样/双通道的一致性,不信模型自报数字 | LLM 自报置信度无校准 |
| 9 | 示意图:AI 填参数、程序画图;电路/光路禁止 AI 徒手画;内测可贴真题截图 | 几何精确性只有确定性渲染器能保证 |
| 10 | 骨架版范围收窄为一个内容切片的完整闭环,不追求全科覆盖。**切片已改选:选择性必修第三册第 1-3 章(分子动理论/气体固体液体/热力学定律)**,以用户提供的 175 页复习讲义为内容基准("讲义内全部内容都要考")。热学是理想首切片:前置依赖少(可独立成环)、图形需求低(p-V 等图像 + 少量装置图)、题型模式化程度高 | 教研工时是硬约束;用户 2026-07-02 拍板换切片(原为力学第一梯队,路线图中"力学三单元"等表述以本条为准) |

## 第四部分 · 风险清单(按严重度排序,全部要在对应时点处理)

### R1 · 教研门槛——全项目最大单点风险
所有质量兜底(边界审核、rubric、选题、审卡)最终裁决者是你,而"某考法在哪些省考到什么深度"连一线教师都常拿不准。**对策(骨架版就要做)**:①引入外部锚点——2025 修订版课标官方 PDF、教育部考试院年度《高考试题分析》、目标省考试院文件、成熟教辅(如五三)的考点边界,审核时对照而不是凭感觉;②找一位在职/退休高中物理教师**兼职抽审**(哪怕只抽 boundary 字段的 10%,哪怕付费),作为人审的仲裁层;③你的审核聚焦在 AI 最容易错的字段(boundary、前置关系、易错点),其余字段抽审+实测兜底。

### R2 · Claude API 的大陆可用性
中国大陆不在 Anthropic 官方支持地区,注册、支付、连接、服务条款都有现实障碍——"自测零障碍、上线才换发动机"这个假设不成立。**对策**:①开发与自测阶段用你现有的可用渠道,但把"密钥失效/不可用"当作已知风险准备预案;②把 Claude 的定位重心放在**教研生产线**(起草卡片、出题、写解析、建测试集——离线内容生产,质量最高、合规风险最低);③运行时引擎严格走适配层,**在接真实学生之前**(不是商业化之前)完成国产模型(DeepSeek/Qwen/GLM 等)的切换评估:切换不是改配置就完——提示词要重调、全套对抗测试集要重跑,预算按"数天到两周"计。

### R3 · 合规红线(商业化前必须逐项过律师,自测阶段不触发)
- **双减政策对高中学科类培训"参照执行"**(资质、非营利化、时段价格监管)+ 教育部明令下线"拍照搜题"类功能——本产品"拍照→批改→讲解"离这条红线很近。**"苏格拉底式不直接报答案"恰是最有利的合规论据,要作为产品设计原则写死**,商业化前先取得"AI 学习工具 vs 学科类校外培训"的法律定性。
- 生成式 AI 服务备案 + 算法备案(实践上意味着面向公众的推理必须用已备案国产模型)。
- 未成年人保护:监护人同意、防沉迷、消费限制;学生数据属未成年人个人信息,发境外 API 构成数据出境场景。
- ICP 备案、教育 App 备案(若做成 App)。
- 版权:内测自用真题风险低;商用安全区 = 自编/改编题 + 自有解析层;"改编"也不当然免责,商用前过律师。
- **自测阶段你只需守两条纪律:API 密钥不进 git;不用真实学生的数据测试。**

### R4 · 批改可靠性
行业头部(学而思被投诉"把对判成错")都没完全解决。我们的设计(标答注入+代码判数值+分步作答+一致性置信+申诉冻结)是当前最强组合,但**误判仍会发生**——产品靠"低置信降措辞+申诉+人工抽检"消化,不靠假装不会错。把对判错是最伤信任的失败模式,金标集要尽早混入真实错误样本(你扮学生写错解),内测第一周重点盯这里。

### R5 · 工时现实(已按审查意见上修 2-3 倍)
五份设计稿的原始工时估算普遍乐观。真实量级见第五部分——这也是决策 #10(收窄到力学第一梯队)的直接原因。**教研工时是这个项目的配速器,不是工程。**

### R6 · 知识点编号的演化
学生所有记录挂在知识点 id 上,而首轮粒度判断必错一批(拆卡/合卡必然发生)。第一天就设计 id 别名与记录迁移机制,"编号永不更改"改为"编号永不复用 + 旧号自动重定向"。

## 第五部分 · 路线图(里程碑制,每步结束都有可实测的东西)

> 节奏假设:你每天投入 2-3 小时教研;工程由 Claude Code 完成。时长是诚实估计,教研进度是配速器。

### M0 · 决策与地基(第 1 周)
- **你**:① 购买《普通高中物理课程标准(2017 年版 2025 年修订)》纸质版/获取官方 PDF;② 拍板目标省份;③ 评估能否找到一位物理教师做兼职抽审;④ 确认 Claude API 可用渠道。
- **我**:项目脚手架(Next.js + SQLite + 模型适配层 + 基础聊天界面)、知识库 schema 与校验脚本、卡片阅读版渲染器、审卡工作流工具(改动过的卡自动降级回待审)。
- **验收**:网页能打开,能和带教师人设的 AI 对话;知识库文件格式定稿。

### M1 · 内容地基:课标库 + 力学第一梯队(第 2-5 周)
- 课标"内容要求"约 100 条逐字录入建库(AI 提取、你逐条对照 PDF 核对)+ 学业质量分级终核(5 级→3 级之争以官方 PDF 为准)。
- 知识树全景(约 30 单元/200 知识点,只定名称归属,你审结构)。
- **精做 3 个单元**:运动的描述、匀变速直线运动、相互作用与牛顿运动定律——约 40-50 张知识卡(boundary 全审)+ 力学模型卡 10-15 张(传送带/板块/连接体…)+ 覆盖率矩阵跑通。
- 全局超纲红线卡 + 50 问钓鱼测试集。
- **验收**:3 个单元覆盖率 100%;钓鱼测试通过率报告;你对任一张卡的阅读版说"这像我认可的教案"。

### M2 · 讲课闭环(第 4-7 周,与 M1 后半并行)
- 卡片检索注入、苏格拉底状态机(诊断→三级提示→解析→变式检验→降阶补基)、三道超纲防线、计算器工具、每轮日志。
- **验收:你扮演学生,被它完整教会"传送带模型"一节,全程无超纲、无事实错误、提示不跳级;钓鱼测试重跑通过。**

### M3 · 练习与批改闭环(第 7-11 周)
- 题库 150-300 道(力学三单元,你选题、AI 结构化录入、你终审)+ 录题工具(优先级不低于学生端)。
- KaTeX 呈现、参数化图形库首批 4 场景(函数图像/水平物块/斜面/传送带)、分步作答、三级提示、代码判分 + 标答注入批改 + 错因标签、错题本双视图 + 遗忘曲线复练(模板级数字变式 + 程序验算)。
- 批改金标集 50 份并入回归。
- **验收**:一次完整练习会话(6-8 题)全流程走通;金标集批改一致性达标;"我认为批错了"通道可用。

### M4 · 记忆与计划闭环(第 11-13 周)
- 学生档案/掌握度/会话摘要注入;摸底诊断卷;每日计划规则引擎 + 每日一次归因调用;停车场机制(超前问题存档、到点兑现)。
- **验收:关掉浏览器第二天回来,它记得你昨天哪里薄弱、主动给出今天的计划——这是产品定义的时刻。**

### M5 · 文件上传(第 13-15 周)
- 粘贴文字 + 清晰照片单题 → 考点映射 → 置信分层 → 确认页 → 薄弱信号入库 → 生成修复计划;映射黄金集(力学 100-150 题)。
- **验收**:上传 5 道真实错题照片,生成的修复计划让你作为"老师"认可其顺序与理由。

### M6+ · 扩展(顺序待 M5 后再定)
电磁学板块(高考分值第二大头)→ 电路/电磁场图形场景 → 整卷 PDF → 真实学生内测前置项(国产模型切换评估 + 合规定性 + 拍照手写转写确认流)→ 内测 3-5 名真实学生。
