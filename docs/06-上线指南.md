> 代码侧的上线准备已全部就绪:多用户访问码、生产构建、Dockerfile、render.yaml。
> 真正把网站放上公网,需要**你名下的两个免费账号**(我无法替你注册),之后的操作我可以一步步带你做。

## 一、还差什么(只差账号)

| 需要 | 用途 | 费用 |
|---|---|---|
| GitHub 账号 | 存放代码,托管平台从这里拉取 | 免费 |
| 托管平台账号(下面二选一) | 实际运行网站,给你一个 https 网址 | 免费层可用 |

**平台怎么选:**
- **Zeabur**(推荐给你):对中国大陆访问快,界面有中文,支持支付宝(免费额度够自测)。
- **Render**:老牌免费层;缺点是免费实例闲置会休眠(首次打开等 30 秒),国内访问偶尔慢。
- 将来正式商用(几百上千学生)再迁移到国内服务器 + ICP 备案,那是另一阶段的事。

## 二、上线步骤(约 20 分钟,我带你走)

1. **注册 GitHub**(github.com,邮箱即可)→ 新建一个空私有仓库(名字如 `learnapp`)。
2. **把代码推上去**:告诉我你的仓库地址,再在 GitHub 生成一个 Personal Access Token 交给我,我来推送;或者你装 GitHub Desktop 自己点两下。
3. **注册 Zeabur / Render** → "从 GitHub 导入项目" → 选中 `learnapp` 仓库,平台会自动识别配置(render.yaml / Dockerfile)。
4. **在平台上填两个环境变量**(不要写进代码):
   - `ANTHROPIC_API_KEY`:你的 key
   - `ANTHROPIC_BASE_URL`:中转平台的接口地址(官方 key 则留空)
5. 部署完成,得到 `https://xxx.zeabur.app` 之类的网址——发给任何学生即可用,各自输入自己的访问码。

## 三、上线后要知道的三件事

1. **学习数据的持久性**:免费托管层的磁盘是"临时"的——平台重启/重新部署时 `data/students/` 会清空。自测和小规模试用可接受;要长期保数据,升级平台的持久磁盘(约 $1-7/月)或接免费云数据库,到时我来改。
2. **API 通道**:当前 key 在官方接口验证失败(不是 `sk-ant-` 格式,应为中转平台 key)。**需要你提供中转平台的接口地址**,我配好并实测后,`server/config.json` 的 provider 切到 `anthropic`,Stellairs 就有真大脑了。上线前建议先在本机测通。
3. **内容版权**:讲义题目截图仅限内测/教学试用;商用化前需替换为授权/自编题库(计划书 R3)。
