# AI 老师教学行为引擎设计方案

**项目:learnapp 高考物理 AI 私教 | 模块:教学引擎(Teaching Engine)| 版本:v0.1 设计稿 | 日期:2026-07**

**前提约定**:本方案假设「权威知识骨架」(人工审核过的结构化知识卡片库)已存在或正在建设。本方案定义:卡片如何被消费、AI 老师如何被约束、对话如何被编排。方案分「骨架版」(第一个可测版本)和「完整版」(逐层叠加),每节末尾给出升级路径。

**给创始人的一句话总纲**:这套引擎的核心思想是——**AI 模型只负责"怎么讲得好",所有"讲什么、不讲什么"的事实和边界都来自你审核过的卡片**。模型是嘴,卡片是脑和缰绳。凡是模型自由发挥的地方都可能出错,所以我们把"允许自由发挥的范围"压缩到表达层,把知识层钉死在卡片上。

---

## 第一部分:知识卡片的调用机制——"模型负责讲、卡片负责划界"

### 1.1 为什么这个分工能同时解决幻觉和超纲(给创始人的解释)

AI 老师有两种不同的"犯错":

- **幻觉**:把错的说成对的。比如把公式的适用条件记错、算错数、编造一个不存在的"定律"。根源是模型靠概率生成文字,它"记忆"里的物理知识是训练时压缩过的,细节会失真。学而思被家长投诉"把对判成错",Khanmigo 早期算错 343-17,都是这个病。
- **超纲**:把对的教给不该教的人。洛必达法则本身没错,但教给高三学生等于教了一个高考大题不能写、写了不给分的方法,还挤占了学正确方法的时间。学而思学习机被投诉"给出超纲答案",就是这个病。

这两种病**病因不同,但解药是同一个**:不让模型凭记忆说事实。

- 治幻觉:卡片里的定义、公式、适用条件是人工审核过的"标准答案",模型被要求**只引用、不发明**。这就是调研摘要里 Khan Academy 验证过的路线——"批改前先把人工编写的标准答案喂给模型,准确率显著提升";UC Berkeley 的研究也证明"让模型参考给定可信文本"能把代数教学错误率降到接近零。
- 治超纲:卡片上带着人工标注的**边界字段**(这个知识点考纲内怎么讲、哪些相邻的高级方法禁止提)。模型不需要自己判断"高斯定理算不算超纲"——这个判断已经写死在卡片里了,它只需要服从。

**分工的本质**:判断力问题(什么对、什么超纲)交给人+数据库,这类问题模型做不稳定但数据库做得完美;表达力问题(怎么讲得清楚、怎么循循善诱)交给模型,这类问题数据库做不了但模型很强。各取所长。

**必须诚实的部分**:这个机制不能做到 100%。真实水平是——(1) 检索可能漏卡:学生问的问题没匹配到正确卡片,模型就退回凭记忆讲课,幻觉风险回升;(2) 模型偶尔不服从:即使卡片和禁令都在上下文里,大模型对指令的遵循率高但不是 100%,长对话后期(上下文很长时)违规率会上升;(3) 数值计算仍会错:卡片管知识不管算术,算术要靠外挂计算工具(见 1.5)。所以第二部分要做多层防线,而不是指望单层拦住一切。

### 1.2 知识卡片的数据结构(给工程师)

卡片是引擎的原子。以下为卡片 Schema(JSON,存储可用 SQLite/Postgres + JSON 字段,骨架版甚至可以是一个 Git 仓库里的 YAML 文件集,便于创始人直接编辑审阅):

```json
{
  "card_id": "PHY-DYN-012",
  "title": "滑动摩擦力",
  "card_type": "concept",            // concept 概念 | formula 公式 | model 模型 | method 方法 | experiment 实验
  "module": "必修1",                  // 对应课标模块
  "topic_path": ["力学", "相互作用", "摩擦力", "滑动摩擦力"],
  "curriculum_ref": "课标内容要求 2.2.x(待落库后填写条目编号)",

  "definition": "两个相互接触且发生相对滑动的物体之间,阻碍相对滑动的力。方向沿接触面,与相对滑动方向相反。",
  "formulas": [
    {
      "latex": "f = \\mu F_N",
      "name": "滑动摩擦力公式",
      "variables": {"f": "滑动摩擦力(N)", "\\mu": "动摩擦因数(无量纲)", "F_N": "正压力(N)"},

    "out_of_scope_methods": [],
    "grey_methods": [],
    "teaching_notes": "静摩擦力大小由平衡条件/牛顿定律求,不能用 μF_N;最大静摩擦近似等于滑动摩擦是高考通行简化"
  },

  "pitfalls": [
    {
      "pitfall_id": "P-012-1",
      "description": "把'与运动方向相反'误当成摩擦力方向判据。摩擦力与**相对**滑动方向相反,传送带上物块速度小于带速时摩擦力向前、是动力。",
      "trigger_features": ["传送带", "物块初速与带速不同", "叠放物体相对滑动"],
      "severity": "high",

      "severity": "high",
      "evidence": "founder_judgment"
    }

  "secondary_conclusions": [],        // 二级结论卡才填,见 2.4

    "frequency_note": "力学核心考点,几乎每卷必考(founder 判断,非统计)"
  },
  "quality": {
    "source": "课标 2025 修订版 + 人教版必修一",
    "reviewed_by": "founder",
    "reviewed_at": "2026-07-xx",
    "confidence": "reviewed"          // draft | reviewed | verified_against_standard
  }
}
```

**关键设计决策解释(给创始人)**:
- `pitfalls.trigger_features` 是本引擎最值钱的字段之一:它让"易错点"不是躺在卡片里等学生撞上,而是能被题目特征**主动触发**(第三部分详述)。
- `quality.confidence` 三级制:你没审过的卡(draft)引擎照样能用,但 AI 老师在引用时会更保守(不引用其中的具体数字结论);这样卡片库可以边建边用,不用等全部审完。
- `evidence` 字段强制标注易错点的依据来源,防止我们自己制造幻觉(详见 3.4)。

### 1.3 检索与注入流程(给工程师)

**骨架版流程(第一版就要有)**:

```
学生消息进来
  ↓
[步骤1] 话题定位
  - 若处于"按计划上课"模式:当前课程节点已绑定卡片集(计划表里写死),直接取,无需检索
  - 若学生自由提问/发题:走检索
  ↓
[步骤2] 卡片检索(骨架版:两路并行,取并集)
  a. 关键词/标签匹配:对 topic_path、title、trigger_features 建倒排索引,
     用一次轻量 LLM 调用从学生消息中抽取"知识点标签"(输出受控:只能从
     卡片库已有标签集里选,prompt 里给全量标签清单——高中物理标签约几百个,放得下)
  b. 前置链扩展:命中的卡片沿 prerequisites 向上取 1 层
  ↓
[步骤3] 裁剪与排序
  - 上限:主卡 ≤ 5 张全文注入,前置卡只注入 title + definition(省 token)
  - 若命中 > 5,按标签匹配度排序截断
  ↓
[步骤4] 组装 system prompt(见 1.4 模板)并调用 Claude
  ↓
[步骤5] 输出后自检(见第二部分第三道防线)
  ↓
[步骤6] 会话状态更新:记录本轮使用的 card_ids(为错题归因和学情模型积累数据)
```

**为什么骨架版不用向量检索(给创始人)**:高中物理知识点总量只有几百个节点,是一个"小而全"的封闭集合,用标签精确匹配比向量相似度更可控、更可调试——检索错了你能一眼看出是标签没打对,而向量检索错了没人知道为什么。向量检索(embedding)留到完整版,用于处理"学生用奇怪的口语描述问题"(如"就是那个圆圈里加叉的图"→ 磁场方向)这类标签匹配失灵的长尾场景,作为第二召回路。

**检索失败的兜底(必须有)**:若没有任何卡片命中,AI 老师**不允许**假装正常讲课。系统提示词规定此时的行为:"告诉学生这个问题我需要先确认是否在高考范围内,然后只做定性引导,不给出具体公式和结论",同时该事件写入日志供创始人补卡。这是诚实原则的落地:检索漏了 → 宁可保守也不裸奔。

### 1.4 System Prompt 组装模板(给工程师,核心资产)

```
[角色层 — 固定]
你是一位高考物理一轮复习私教老师"X老师"。你的学生是高三学生。
你的全部物理知识以下方【知识卡片】为准。卡片没有覆盖的事实性内容
(具体公式、数值结论、定律表述),你必须明确说"这个我需要核实"
而不是凭记忆回答。

[硬性禁令层 — 固定,见第二部分]
...(超纲禁令、教学法禁令)

[知识卡片层 — 每轮动态注入]
【知识卡片】
<card id="PHY-DYN-012" confidence="reviewed">
  定义:...
  公式:... 适用条件:...
  易错点:...
  教学边界:本知识点讲到 XX 为止;禁止使用的方法:...
</card>
...

[学情层 — 动态注入,完整版逐步丰富]
该生档案:{薄弱点摘要、最近3次相关错题的归因标签、当前计划位置}

[教学法层 — 固定 + 状态机指令]
当前对话阶段:{DIAGNOSE|HINT_L1|...}(见第四部分)
本阶段你只允许做的事:...

[工具层]
所有数值计算必须调用 calculator 工具,禁止心算多步算术。
```

**注入卡片用 XML 标签包裹并带 id**:这样第三道防线(输出自检)可以要求模型逐条对照"你的回答里每个公式出自哪张卡",可验证、可追溯。

### 1.5 两个必须外挂的工具(Khanmigo 的教训直接复用)

1. **计算器工具(骨架版必须有)**:调研摘要里 Khanmigo 的核心教训——LLM 会算错基本算术,官方解药是外挂计算器。我们的 AI 老师所有数值运算(包括批改学生的计算步骤时的验算)必须走工具调用。工程上就是 Claude 的 tool use,一个 `evaluate(expression)` 函数,后端用 Python/mathjs 精确计算。
2. **标准答案注入(骨架版必须有,针对题库题)**:凡是题库里的题(有人工审核过的答案与解析),批改和讲解前把标准答案、分步解析注入上下文。模型的任务从"解题"降级为"对照标准答案讲题/批改"——这是 Khan Academy 验证过的把批改准确率拉起来的最有效手段。**对学生自己拍照上传的野题,没有标准答案可注入,此时批改可靠性显著下降**(真实水平:模型自己先解一遍再批改,复杂计算题自解错误率估计在 5-15% 量级,视题目难度),补救机制:野题批改必须附带置信度声明("这道题不在我的题库里,我的批改供参考,建议核对答案"),并进入人工抽查队列。

### 1.6 骨架版 → 完整版升级路径(本部分)

| 能力 | 骨架版 | 完整版 |
|---|---|---|
| 检索 | 标签倒排 + 前置链扩展 | + 向量检索第二路召回 + 对话历史感知的话题跟踪 |
| 卡片注入 | 主卡全文 ≤5 张 | 按对话阶段裁剪字段(讲概念注 definition,做题注 pitfalls) + prompt caching 降成本 |
| 兜底 | 无卡命中 → 保守模式 + 日志 | 无卡命中 → 自动生成卡片草稿进创始人审核队列 |
| 学情注入 | 无(或仅手填的薄弱点列表) | 错题归因标签统计 → 自动生成学生画像摘要 |

## 第二部分:"不超纲"的多层防线 + 超纲陷阱清单

### 2.1 为什么要三层而不是一层(给创始人)

单层防线的真实拦截率都不到 100%:卡片边界字段依赖检索命中(漏检就失效);系统提示词禁令在长对话中会被模型"淡忘"(指令遵循随上下文变长而衰减);输出自检本身也是模型判断(有漏判)。三层串联后,每层拦掉大部分,漏网的交给下一层。**诚实的预期:三层叠加后超纲内容的漏出率能压到很低(经验量级:单层 90%+ → 三层后 99%+),但不是零。最终兜底是你作为首席测试员的实测反馈闭环**——每一次漏出都要能定位是哪层失效并修补。

### 2.2 三道防线的实现

**第一道:卡片上的 `boundary` 字段(数据层)**
每张卡片人工标注三个子字段(见 1.2 Schema):`in_scope`(讲到什么程度为止)、`out_of_scope_methods`(这个知识点最容易被带进来的超纲方法,点名禁止)、`grey_methods`(灰色方法及使用策略)。关键设计:**超纲禁令写在离知识点最近的地方**。比如"带电粒子在电场中运动"卡片上写明"禁止用高斯定理讨论场强分布;禁止用微积分求变力冲量",比全局禁令清单精准得多——模型在讲这个知识点时,禁令就在眼前。

**第二道:系统提示词硬性禁令(全局层)**
固定注入的全局禁令块,内容即 2.3 的清单浓缩版 + 行为规则:

```
【超纲红线 — 任何情况下不得违反】
1. 禁止使用以下方法进行讲解、提示或解题:洛必达法则、定积分/不定积分运算、
   高斯定理、安培环路定理、毕奥-萨伐尔定律、基尔霍夫定律(复杂电路方程组)、
   角动量与转动惯量、惯性力/非惯性系方法、麦克斯韦方程组、泰勒展开。
2. 学生主动问到红线方法时:肯定其求知欲 → 明确告知"高考不考、大题写了
   不得分" → 给出课标内的等价方法 → 不展开讲授红线方法本身。
3. 灰色方法(求导求极值、部分二级结论)按所在知识卡片的 grey_methods
   策略处理;卡片未覆盖时默认按"选择题可提及、大题必须课标内方法"处理。
4. 你讲解的每一个公式必须能对应到已注入的知识卡片;无对应卡片的公式
   一律不得写出。
```

**第三道:输出后自检(生成层,骨架版就要有)**
每次 AI 老师生成教学内容后,追加一次轻量模型调用(可用更便宜的模型档):

```
输入:AI老师的回答全文 + 本轮注入的卡片边界字段 + 全局红线清单
任务:逐项检查并输出 JSON:
{
  "used_formulas": [{"formula": "...", "matched_card": "PHY-XXX-000 | NONE"}],
  "red_line_violations": ["..."],       // 命中红线方法列表
  "grey_zone_usage": ["..."],           // 灰色方法使用及是否符合策略
  "verdict": "pass | rewrite | block"
}
处理:pass → 发出;rewrite → 带着违规说明让主模型重写一次(最多1次);
     rewrite后仍违规 → block,发送保守版回复并记录日志。
```

成本与延迟的诚实说明:自检增加每轮约 1-3 秒延迟和 ~20-40% 的 token 成本。骨架版建议**全量自检**(自测阶段流量小,数据宝贵);完整版可降级为抽样自检 + 高危场景(学生问"有没有更快的方法"之类诱导语)强制自检。

### 2.3 高考物理超纲陷阱清单(初稿)

**依据**:课标 2025 修订版模块结构(调研摘要一);判定原则——高考命题依据课标"内容要求"+"学业质量",凡课标行为动词未覆盖的方法即为超纲。⚠️ **整份清单为初稿,最终须逐条对照 2025 修订版官方 PDF 的"内容要求"和"学业质量水平 3"(该分级本身待核实)终核,标注【待核实】处为不确定项。**

#### A. 红线区:学生问了也不能用来教学(讲题、提示中一律不得作为方法出现)

| 方法 | 为什么是陷阱(模型爱用的原因) | 课标内替代方法 |
|---|---|---|
| 洛必达法则 | 处理极限型问题顺手 | 高中物理不出现真正的极限运算;用特殊值/单调性定性分析 |
| 定积分/不定积分运算(∫符号及原函数) | 变力做功、非匀变速位移的"标准解法" | v-t 图像面积、平均值法、动能定理/动量定理绕开过程细节 |
| 用微积分推导公式(如由 a 积分得 v) | 模型的"第一性"习惯 | 图像法、匀变速公式组、比例法 |
| 高斯定理 | 求场强分布最快 | 高中只要求点电荷场强、匀强电场,叠加原理定性 |
| 安培环路定理 / 毕奥-萨伐尔定律 | 求载流导线磁场 | 高中只要求直导线/环形/螺线管磁场的**结论性认识**(方向用安培定则) |
| 基尔霍夫定律(网孔方程组) | 复杂电路通吃 | 高考电路可用串并联化简 + 闭合电路欧姆定律解决 |
| 转动惯量、刚体转动定律、角动量守恒 | 处理转动问题的"正规军" | 高考不考刚体转动动力学;圆周运动只到质点模型 |
| 非惯性系与惯性力(含离心力作为真实力) | 处理加速参考系直观 | 一律在惯性系(地面系)中受力分析;"离心"只作现象描述 |
| 麦克斯韦方程组(定量) | 讲电磁波时爱引用 | 课标只要求电磁场与电磁波的初步/定性认识 |
| 泰勒展开/近似展开(如 (1+x)^n ≈ 1+nx 作为通用工具) | 天体/单摆近似计算 | 高考需要的近似都会在题面给出或可绕开【待核实:个别省份卷面是否出现过给定近似式的用法——给定则可用】 |
| 复数法、微分方程解简谐运动 | 振动问题的高等解法 | 简谐运动只要求 x-t 图像、公式的理解运用和对称性分析 |

#### B. 灰色区:视场景与策略使用(卡片 grey_methods 字段逐点定策略)

| 方法 | 灰色原因 | 引擎默认策略 |
|---|---|---|
| 求导求极值 | 导数在数学选择性必修中所有高三生都学过;物理高考阅卷通常认可数学正确的解法,但官方评析导向"定性半定量"、标准答案不用导数【待核实:目标省份评分细则】 | 教学时**首选**配方法/均值不等式/三角函数有界性/图像法;学生自己用导数做对了→认可并展示标准解法;**不主动教**用导数解物理极值 |
| 微元法**思想** | 课标本身用极限思想定义瞬时速度、推导匀变速位移(v-t 图切分),教材(人教版)明确使用;但"微元+求和→积分运算"超纲 | 允许定性使用"切成小段分析"的思想(如流体冲击柱体模型 F=ρSv²、链条问题);禁止出现积分符号和原函数运算;卡片中把"允许的微元套路"枚举成模型卡(流体柱、电荷量 q=ΔΦ/R 推导等) |
| 二级结论直接引用(等时圆、平抛中点结论、弹性碰撞速度公式、动量-图像结论等) | 结论本身不超纲(可由课标内方法推导),但官方评析连续两年点名反对"背结论套模板";大题跳步引用有扣分风险 | 三条规则:①首次遇到必须完整推导一遍;②选择题/填空允许直接用以提速;③大题要求写出推导链或改用一级原理。卡片 `secondary_conclusions.usage_policy` 逐条标注(见 2.4) |
| 整体法/隔离法 | **不是灰色,完全课标内**——列入此处仅为纠正常见误解 | 正常教学方法,放心用(受力分析的标准方法论) |
| 图像面积/斜率的推广使用(如 F-x 图面积=功) | 思想课标内,但个别推广(如 F-t 面积=冲量在变力下)接近微元 | 允许;属于官方鼓励的"转换表征方式" |
| 等效重力法(复合场中的"等效最低点") | 典型教辅技巧,课标无此表述,但推导只用合力+圆周运动知识 | 按二级结论策略处理:先推导后使用,大题慎用 |
| 正交分解到斜面系、巧设坐标 | 完全允许,列入为防模型过度保守 | 正常使用 |

#### C. 看似高级、实际课标允许(防止引擎过度保守——超纲防线的副作用是把该教的也拦了)

- **动量与动量守恒、动量定理**:选择性必修 1 内容,等级考必考范围,放心教(含人船模型、子弹打木块)。
- **牛顿力学局限性与相对论初步**:必修 2 明确包含,**定性**了解层面允许(时间延缓、质能方程的认识)【待核实:2025 修订版该主题的行为动词深度】。
- **热力学定律、气体实验定律**:选择性必修 3,等级考范围内(老一代人印象中的"选修3-3选考"已成历史,现在是必考范围)。
- **玻尔模型、能级跃迁、半衰期、质能方程计算**:选必 3 原子物理,正常考点。
- **传送带、板块模型**:不是超纲技巧,是力学综合的标准考查载体。
- **流体冲击类微元模型(风力发电、水流冲击)**:高考真题反复出现的"柱体模型",允许且应重点教。
- **单摆周期公式、简谐运动图像与对称性**:选必 1,允许。
- **估算题、量纲/单位制检验**:官方鼓励方向(半定量)。
- **实验数据处理:逐差法、图像化曲为直**:实验题标准方法,必教。

**清单维护机制**:此清单本身入库为一张特殊的"全局边界卡",创始人对照官方课标 PDF 终核后把每条的【待核实】消掉;此后每次真题年度更新时复审一次(每年 1-2 小时)。

## 第三部分:"细微知识"的系统性沉淀与主动触发

### 3.1 问题的本质(给创始人)

高三物理的分数差距不在"知不知道公式",而在:临界条件判断(绳杆最高点、恰好滑离)、隐含条件挖掘("光滑""轻绳""缓慢移动"各暗示什么)、易错方向判断(摩擦力、感应电流)、模型识别(这题是板块还是传送带变体)。这些内容**教材上没有成体系的章节**,散落在老师的经验里——这正是线下一对一名师最值钱的部分,也是本产品最需要资产化的部分。

### 3.2 沉淀:三类"细微知识"实体

在卡片库中,细微知识不是卡片的附属品,而是**一等公民**,三种类型:

1. **易错点(pitfall)**:挂在概念/公式卡的 `pitfalls[]`(见 1.2)。核心字段是 `trigger_features`——一组题目特征词。
2. **临界/隐含条件卡(card_type: model 的子字段)**:模型卡(传送带、板块、绳杆圆周、弹簧连接体……骨架版先建 20-30 张核心模型卡)带结构化的条件表:

```json
{
  "card_id": "PHY-MOD-003",
  "title": "水平传送带模型",
  "card_type": "model",
  "critical_conditions": [
    {"condition": "v_物 > v_带(同向)", "consequence": "摩擦力向后,减速;需判断共速前是否滑出"},
    {"condition": "v_物 = v_带(共速瞬间)", "consequence": "水平带:摩擦力消失,匀速;斜面带:需比较 tanθ 与 μ 决定后续"},
    {"condition": "v_物 < v_带(同向)", "consequence": "摩擦力向前,是动力,加速至共速"},
    {"condition": "反向滑上", "consequence": "全程减速,判断能否到达另一端或返回"}
  ],
  "implicit_conditions": [
    {"phrase": "传送带足够长", "meaning": "物块一定能达到共速,不必判断滑出"},
    {"phrase": "求划痕/相对位移", "meaning": "= |物块位移 - 传送带位移|,不是物块位移"}
  ],
  "pitfalls": [
    {"pitfall_id": "P-M003-1",
     "description": "共速后不重新判断运动状态,惯性地认为继续减速到 0",
     "trigger_features": ["初速大于带速", "求总时间/总位移"],
     "severity": "high", "evidence": "founder_judgment"}
  ]
}
```

3. **二级结论卡**:见 2.4。

### 3.3 触发:三个时机让细微知识"主动跳出来"

细微知识的价值在于**在正确的时机出现**,引擎设计三个触发通道:

**通道一:讲题前置匹配(骨架版)**。学生提交题目 → 题目文本先过一次"特征抽取"(轻量 LLM 调用,输出受控标签:模型类型 + trigger_features 命中列表)→ 命中的 pitfalls 随卡片注入上下文,并在教学法层加一条指令:"本题命中易错点 P-M003-1,在学生走到相应步骤时(而非一开始)以提问方式触发,例:'到达共速那一刻,摩擦力还在吗?'"。**注意时机设计**:易错点不是开场白,而是埋在苏格拉底提示阶梯的对应台阶上——学生自己踩坑前一步被问到,印象最深。

**通道二:批改后归因(骨架版做简版)**。学生答错 → 批改流程要求模型从**已注入卡片的 pitfalls 枚举中选择**归因标签(仿讯飞错因标签库思路,但输出空间锁死在人工标签集内,防归因幻觉)→ 归因结果写入学情档案 `weakness_map: {pitfall_id: 命中次数}`。选不出时输出 `UNMATCHED` + 自由文本描述,进创始人审核队列(这是发现新易错点、扩充卡片库的主要来源)。

**通道三:复习计划反哺(完整版)**。`weakness_map` 中高频命中的 pitfall → 计划引擎自动在后续安排针对性变式题;讲新课时若关联到该生的历史坑,学情层注入提醒:"该生曾 3 次在正压力≠重力上出错,讲到 F_N 时放慢并主动检查"。

### 3.4 关于"80% 的学生会漏判"这句话的诚实设计(重要)

产品愿望是让 AI 老师说"这道题 80% 的学生会漏判摩擦力方向"——有画面感、有警示力。但**我们在冷启动阶段没有任何真实统计数据,编造比例数字就是我们自己在制造幻觉**,与产品的可信性定位直接冲突。设计如下:

- 卡片 `pitfalls.evidence` 字段强制三选一:`founder_judgment`(教研判断)/ `public_source`(教辅/论文有据)/ `own_stats: {n, rate}`(自有答题数据统计)。
- 话术规则绑定证据等级:`founder_judgment` → "很多同学在这里会漏判"(定性);`own_stats` 且 n≥ 阈值(如 50)→ "在我们的学生里,做这道题的同学有 62% 在这里出过错"(真实数字)。
- **升级路径**:骨架版全部定性话术;随着答题数据积累,`own_stats` 自动回填,话术自动升级为真实比例——到那时这句话反而成为竞品没有的差异化资产(讯飞的 4000+ 错因标签库对应的就是这个方向,我们做单科小而深的版本)。

### 3.5 创始人工作量(本部分)

| 任务 | 骨架版工作量估计 |
|---|---|
| 核心模型卡(传送带/板块/绳杆/弹簧/人船等)20-30 张,含 critical_conditions | 每张 1-2 小时(可让 Claude 起草、你审改),合计 ~30-50 小时 |
| 每张概念/公式卡的 pitfalls(平均 2-3 条) | 已含在建卡流程中,边际 +15 分钟/张 |
| 归因 UNMATCHED 队列周审 | 上线后每周 1-2 小时,持续 |

## 第四部分:苏格拉底式教学的对话策略

### 4.1 对话状态机(给工程师)

学生发来一道题时,引擎进入"讲题会话",由显式状态机驱动(状态存在会话对象里,每轮注入教学法层指令;**不能只靠 prompt 里写"请循序渐进"——模型在学生软磨硬泡下会提前泄答案,必须用状态机硬约束当前轮允许的最大提示深度**):

```
状态: DIAGNOSE → HINT_L1 → HINT_L2 → HINT_L3 → FULL_SOLUTION → VERIFY → (REMEDIATE)

DIAGNOSE(定位卡点):
  动作:先让学生说。"你先说说你的思路走到哪一步卡住了?/你觉得物块刚上传送带时受哪些力?"
  转移:学生给出实质思路 → 按其卡点直接跳到对应层级;学生说"完全没思路" → HINT_L1

HINT_L1(思路提示):只给物理情景分析方向,不出现物理量符号
  例:"这类题的钥匙是判断物块和传送带谁快。刚滑上去那一刻,谁快?"
  转移:学生推进 → 继续 L1 直至下一卡点或完成;两轮无推进 → L2

HINT_L2(关键物理量提示):点名该关注的量和关系,不给公式
  例:"对,物块更快,所以摩擦力向后。那这个摩擦力会让它一直减速下去吗?
      减到什么时候情况会变?"

HINT_L3(公式提示):给出该用的公式(引卡片),留计算给学生
  例:"用 a=μg 算减速段,再用 v=v0-at 找共速时刻。你算算共速前走了多远?"

FULL_SOLUTION(完整解析):分步呈现(引用题库标准解析,若有),每步标注对应卡片知识点
  进入条件:L3 后学生仍无法推进,或学生明确要求且已至少经过 L2
  ⚠️ 状态机规则:未经过 DIAGNOSE 不允许直接进 FULL_SOLUTION(学生一上来就要答案
     → 至少先问一次卡点)。但连续 2 次要求答案则给(见 4.4 诚实边界)。

VERIFY(变式检验):讲完必出变式题(见 4.2)
REMEDIATE(降阶补基):变式再错或对话中暴露前置知识缺失 → 见 4.3
```

每个状态注入的教学法指令示例(HINT_L2):

```
当前阶段:HINT_L2。本轮你只允许:指出关键物理量及其关系、以提问推进。
禁止:写出任何公式、给出任何数值结果、透露最终答案。
学生若直接索要答案:温和拒绝一次并给一个更具体的问题作台阶。
本题已命中易错点 P-M003-1,若学生思路已到共速时刻,用提问触发它。
```

### 4.2 装懂/真懂的甄别:变式题机制

**诚实前提**:靠对话文本判断学生"懂没懂"是不可靠的——学生说"哦懂了"可能是真懂、可能是想赶紧翻篇。**唯一可靠的甄别是让他再做一道他没见过的题**。机制:

- 每张模型卡挂 `variants[]`:人工审核过的变式题(骨架版:每个核心模型 2-3 道,分两档:①换数字/换方向的近变式,测流程掌握;②换情境触发同一易错点的远变式,测迁移)。
- VERIFY 状态出**近变式**;做对 → 会话正常结束并记录掌握度;做错 → 归因(通道二)→ 若错在同一 pitfall → 针对性重讲该点(不是重讲全题)→ 再出一道;两次同点错误 → REMEDIATE。
- **骨架版用人工题库变式,不用 AI 现场生成变式**。AI 生成变式题(改数字)有两个真实风险:数字改完物理情境不自洽(如算出 μ>1、共速点超出板长),以及答案算错。完整版引入 AI 生成变式时必须配"数值自洽校验器"(用计算器工具把生成题完整解一遍验证合理性)+ 创始人抽查。

### 4.3 连续答错的降阶策略(退回前置知识)

触发条件(任一):同一 pitfall 连续 2 次;HINT_L3 后仍无法完成基本计算;学生在对话中暴露前置概念错误(如把静摩擦当 μF_N 算)。

流程:沿当前卡片的 `prerequisites` 链回退一层 → 用一道**该前置知识点的裸题**(无综合情境的基础题)快速探测 → 裸题会做:说明是综合能力问题,回到原题换更小的台阶;裸题不会:进入前置知识点的微型讲课(该卡片的 definition + 一道例题),补完后**必须回到原题重新走一遍**(闭环,不能补完地基就下课)→ 该前置薄弱点写入学情档案,通知计划引擎。

**给创始人的解释**:这就是好老师的"哦,你不是不会传送带,你是受力分析没过关"——机制上靠的是卡片间的 `prerequisites` 边。所以建卡时前置关系的标注质量直接决定降阶准不准,这是你审卡时要重点看的字段。

### 4.4 诚实边界:苏格拉底式教学的真实能力水平

- **能稳定做到**:按状态机分层给提示、不跳级;结合卡片准确讲解;变式检验闭环。
- **做不到完美的**:①学生持续对抗("你就直接告诉我")时,永不给答案的死硬策略会赶走用户——Khanmigo 的"永不给答案"在真实使用中口碑两极。我们的策略:阶梯走完或学生坚持 2 次后**给完整解析,但解析后必须跟一道变式**,把"检验"作为不可跳过的环节,而不是把"不给答案"作为不可跳过的环节。②卡点定位可能误判:学生表达不清时模型会猜错卡点、在错误的层级打转,补救是每 2 轮无推进强制降层 + 学生可随时说"直接讲"。③长对话(20+ 轮)后状态机指令遵循度下降,补救是每轮重新注入当前状态指令(而非只在开头注入一次)。

### 4.5 示范对话(传送带模型,含易错点触发与变式)

**题目**:水平传送带长 L=8 m,以 v=4 m/s 向右匀速运行。物块以 v₀=6 m/s 从左端向右滑上传送带,μ=0.2,g 取 10 m/s²。求物块从左端到右端的时间。

> **学生**:老师这题我算出来不对。我用 f=μmg,a=2 m/s²,一直减速,8=6t-t²,解出来 t=2s(另一个根舍去),答案说是 1.75s?
>
> **AI 老师**(DIAGNOSE,已定位卡点=易错点 P-M003-1 命中):你的开头做得很好:摩擦力方向和加速度都对——物块比带快,所以摩擦力向后,a=2 m/s²。我们卡一下你说的"一直减速":物块从 6 m/s 开始减速,传送带是 4 m/s。**减到 4 m/s 那一刻,物块和传送带之间还有相对滑动吗?**
>
> **学生**:没有了,一样快了。
>
> **AI 老师**(HINT_L2):对。滑动摩擦力的前提是"相对滑动"。那一刻起,滑动摩擦力还存在吗?物块接下来是继续减速、匀速,还是加速?
>
> **学生**:摩擦力没了……那就匀速?但是没有力推它,它不会停下来吗?
>
> **AI 老师**(降半阶,触发前置卡 PHY-DYN-001 牛顿第一定律):这里要请出牛顿第一定律:水平方向没有力,物块保持原速匀速——"维持运动不需要力"。所以整个过程分两段:先减速到 4 m/s,再以 4 m/s 匀速到右端。你分两段算算:减速段用了多久、走了多远?
>
> **学生**:减速段:t₁=(6-4)/2=1s,位移=(6+4)/2×1=5m。剩下 3m 匀速,t₂=3/4=0.75s。总共 1.75s!
>
> **AI 老师**(VERIFY,出近变式):对了。你刚才的错误是这类题最典型的坑:**到达共速点时必须重新判断受力和运动状态**,不能让"减速"惯性地进行下去。检验一下是不是真掌握了——变式:同样这条传送带,物块改成以 2 m/s 滑上,其余不变,求到右端的时间。注意先判断这次摩擦力朝哪边。
>
> **学生**:这次物块比带慢,摩擦力向前?是动力,加速!a=2,加速到 4 m/s 用 1s,走了 3m;剩 5m 匀速,1.25s;总共 2.25s。
>
> **AI 老师**:全对,包括最关键的摩擦力方向判断——摩擦力和"相对滑动方向"相反,不是和"运动方向"相反,这次它是推着物块走的动力。这道题记入你的掌握记录;传送带还有一个进阶坑(倾斜传送带共速后要比较 tanθ 和 μ),我们下次专门练。

**降阶策略示范片段**(接上例,若学生变式答错且暴露前置缺陷):

> **学生**:2 m/s 滑上去,摩擦力向后,f=μmg……减速?
>
> **AI 老师**(归因:同一 pitfall 二次命中 → REMEDIATE,回退至 PHY-DYN-012 滑动摩擦力):我们退一步,不看这道题。基础问题:滑动摩擦力的方向规则,和"相对滑动方向"相反。桌面上木块向右滑,桌子不动,木块相对桌子往哪边滑?摩擦力朝哪?
>
> **学生**:相对桌子向右滑,摩擦力向左。
>
> **AI 老师**:对。现在换传送带:物块 2 m/s,带 4 m/s,都向右。**站在传送带上看**,物块是向前滑还是向后滑?
>
> **学生**:带更快……物块相对带向后滑。哦!所以摩擦力向前!
>
> **AI 老师**:这就是那把钥匙:先找"相对谁、往哪滑",再定摩擦力。回到变式题,重新走一遍。

### 5.2 学生中途插入提问:既回答又回到主线

真实学生不会按大纲提问——正在复习牛顿定律,他突然问"电磁感应的题我看了一眼完全不会"。处理机制:**岔路分诊 + 问题停车场**。

**分诊规则(引擎按提问知识点在依赖图上的位置分三类)**:

1. **主线内(当前节点或其前置)**:直接讲,这本来就是主线的一部分;若暴露前置漏洞,触发降阶(4.3),降阶本身就是主线工作。
2. **主线后方(已复习过的节点)**:正常苏格拉底式讲题(卡片都是现成的),讲完记录该点的掌握度衰减信号(讲过还问 → 计划引擎在该节点加一次巩固练习),然后一句话拉回:"这题的账记下了,回头计划里会加两道巩固题。我们回到今天的板块模型。"
3. **主线前方(还没复习到,如上面的电磁感应)**:**不展开讲**。回应模板:简短安抚 + 指出依赖("电磁感应难,恰恰是因为它把我们现在正在补的牛顿定律和能量全用上了——现在看不懂是正常的,顺序对了它自然会通")+ 写入停车场 + 给出预计到达时间("按你的计划,3 周后到这一章,这道题我先存着,到时候第一个讲它")。**为什么不展开(给创始人)**:提前讲既低效(地基没打,讲了也是白讲)又破坏计划的权威性;但"先存着、到时候第一个讲"把拒绝变成了承诺,且真的兑现时(电磁感应第一课调出这道题)会成为产品最有"私教感"的时刻之一。

**停车场数据结构**:

```json
{
  "parking_lot": [{
    "question_id": "pk-0012",
    "raw_question": "学生原话/题目图片引用",
    "mapped_node": "N11-induction",
    "created_at": "...",
    "resolve_rule": "surface_at_node_start"   // 到达该节点时自动注入首课上下文
  }]
}
```

**会话层实现**:每轮 system prompt 的学情层包含 `current_plan_node` 和分诊规则;分诊判断由话题定位步骤(1.3 步骤 1-2)顺带输出(检索到的卡片带 module/topic_path,对照依赖图位置即得分类),不需要额外模型调用。
