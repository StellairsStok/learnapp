OVERALL:
这份方案在同类文档里属于少见的扎实:核心架构决策(闭集“识别+映射”而非自由生成、置信度三档分层、人在环兜底、按失败源逐层上线)方向正确,API 层面的技术论断经逐条核实绝大多数属实(模型ID、定价、1M上下文、2576px视觉、PDF限制、缓存最小前缀、Batch五折均无误),只有 temperature 版本表述一处硬伤。但它有两个未声明的致命前置:一是整个教学出口依赖一个不存在且有版权风险的题库,二是把学生数据送境外 API 的合规问题只字未提——这两件事任何一件卡住,提取管线做得再好也无法变成产品。最薄弱的一环是“权威骨架+黄金测试集”这对地基:两者都要由一个教研能力未经验证的单人创始人,对着一份很可能不存在的“2025修订版课标”独立造出并独任裁判,而方案里所有准确率数字、阈值校准和“有逻辑的教学”承诺,全部悬挂在这对地基之上。建议先解决题库来源与合规路线两个死结、把黄金集扩到有统计意义的规模,再进入管线开发。

FACTUAL_ERRORS:
- 【课标版本高度存疑】方案两处以“2025修订版课标”为骨架锚点(0.1节、kp节点的 curriculum_ref "2025修订版课标 3.2.2")。现行有效的是《普通高中物理课程标准(2017年版2020年修订)》;截至可核实的信息,教育部并未正式颁布高中物理课标的“2025修订版”。若该文件不存在,全部 curriculum_ref 字段和“以课标内容要求为锚点”的权威性叙事都悬空,必须改锚到 2017版2020修订,或由创始人出示 2025 版原文。
- 【temperature 论断不准确】“Claude 4.6+ 系列不接受 temperature 参数”是错的:temperature/top_p/top_k 是从 Opus 4.7 起才被移除并返回 400,Opus 4.6/Sonnet 4.6 仍接受(仅 deprecated)。对方案选定的 claude-opus-4-8 结论碰巧成立,但版本表述错误;更重要的是由此推出的自洽采样方案(“独立请求+轻微改写候选顺序”)把“位置扰动”当成了“独立采样”,三次一致可能只是确定性趋同的假象,而非置信度信号——这一整个置信度地基未经验证。
- 【OCR/多模态准确率数字属伪精确】“Mathpix 文档级公式完全正确率 62.9%”“高难度手写 55% vs 84%”“2025年研究可接受转写率约 88%”均引自无法核查的“调研摘要”,且此类研究基本是英文/数学作业场景;外推到中文潦草手写+物理符号+图文混排的错题本没有依据。第2章整张“真实准确率账本”(85–92%、75–88% 等)全是未经任何实测的区间,却被当作分层阈值和上线判断的依据。
- 【“权威知识骨架”名不副实】高考考试大纲已于2019年底废止(方案未提),课标“内容要求”只有数十条粗粒度条目;150–200个考点节点、前置DAG、难度分层、典型题型全部是 Claude 生成+创始人自审的自建产物。把它称为“人工审核过的权威参照物”夸大了其地位——其权威性上限就是创始人的教研水平,而创始人的教研能力在方案中是未经验证的假设。
- 【教材版本以偏概全】“选择性必修2”“必修3”等模块归属按人教版命名(就人教版而言是对的:电磁感应在选择性必修第二册、闭合电路欧姆定律在必修第三册),但全国存在粤教/鲁科/教科/沪科等多版本教材,模块划分和顺序不同;方案未声明版本假设,骨架 module 字段会对非人教版学生产生错位。
- 【API 论断核实结果——多数正确,应说明以示公平】经查:claude-opus-4-8 真实存在、$5/$25 定价、1M 上下文、视觉长边 2576px、单图最高约 4800 token、PDF 32MB/600页、Opus 4.8 最小可缓存前缀 4096 token、缓存读约0.1倍价、structured outputs 用 output_config.format(旧 output_format 已弃用)、Batch API 五折且结果按 custom_id 无序返回——这些均属实。API 层面的硬伤只有 temperature 一条。

MUST_FIX:
- 【题库是隐藏的死结】第⑥步“从题库按考点取2-3道诊断小题”“3-5题针对练习”——题库在整个方案里不存在,也不在任何工作量清单里。没有按考点标注的题库,“先测后教”“过关判定”整章不可执行;而建题库要么人工录入(单人不可承受),要么爬取/复用真题和教辅题(教育部教育考试院对高考真题主张权利,教辅题目有明确版权,商用有法律风险)。这是比提取管线本身更大的前置工程,必须单独立项并先给出合法来源方案。
- 【中国合规问题完全缺席】把高三学生(多为未成年人或刚成年)含姓名、学校、成绩的试卷照片传给境外 Claude API,直接触及 PIPL 的个人信息出境规制;面向境内公众提供生成式AI服务需按《生成式人工智能服务管理暂行办法》完成备案,教育类应用另有教育APP备案要求。方案把“切国产模型”仅当作技术抽象层的预留,实际上它很可能是上线前的合规硬约束——这会反过来推翻方案里所有基于 Opus 视觉能力的准确率假设(国产模型多模态能力需另行实测),应提前到骨架版决策而非“将来再说”。
- 【黄金测试集在统计上不成立】50–100道题对 150–200 个考点,平均每考点不足1题,既无法给出“主考点准确率85–92%”这种断言,也无法支撑“高置信档实测准确率≥95%反推阈值”的校准;且单人标注、无标注协议、无一致性检验——方案自己承认“两位真人教师都可能意见不一”,却让一个教研能力未经验证的创始人独任裁判。至少需要分板块300–500题、成文标注指南、抽样双标(可请一位在职教师兼职复核)。首建8–15小时的估计不含找题、脱敏、录入、争议裁决,严重偏低。
- 【过度承诺三连,需改写表述】(a)“结构上不可能超纲”:闭集分类只防“发明新考点名”,不防超纲题被强行映射到纲内格子——out_of_scope_flag 本身就是模型判断,会漏;(b)“错误可发现、可纠正、不污染学习计划”:高置信档按其自身目标也有约5%错误直接自动入库进计划,且“默认全部预选确认”的交互会让学生橡皮图章式通过,既没纠错、还把脏标注回流成了“免费标注数据”;(c)“随着数据积累准确率持续爬升”:Claude API 不可微调,回流数据只能用于 prompt/few-shot 人工迭代,提升有限、非自动、且每次改动都要重跑付费回归——“越用越准”是弱承诺,不应写成产品卖点。
- 【schema 缺坐标字段,确认页交互无法实现】切分输出的 question 对象没有 bounding box/页内坐标,但3.2节承诺“第12题(缩略图)”展示和“合并/拆分一键修正”——没有题目在原图中的位置坐标,缩略图裁不出来、拆分合并也无从操作。这是 schema 级缺陷,骨架版就必须补 spans 坐标字段并要求模型输出(而这又会引入新的坐标准确率问题,需纳入测试)。
- 【单人+Claude Code 的运维面被整体忽略】方案默认“回归测试纳入CI”“异步任务队列”“复核后台”存在,但不写代码的创始人无法维护 CI、排查部署故障、管数据库备份和图片存储;每次回归跑黄金集都是真实 API 开销(300题×两阶段×抽样,单次可达数十到上百元),方案未列这笔持续成本。应明确:骨架版一切从“本地脚本+手动跑”开始,CI/队列是奢侈品。
- 【延迟与缓存经济性假设偏乐观】骨架版“单题同步20–40秒”对两阶段 Opus 调用+图片输入偏乐观(Opus 4.8 高 effort 下单请求可达数分钟),移动端前台等待40秒以上基本等于流失;“缓存读1/10价”只在5分钟 TTL 内有连续请求时成立,早期单用户零星上传几乎全是冷缓存(1.25倍写入价),成本应按冷缓存重算,或改用1小时TTL(2倍写入价)重新算账。

NICE_TO_HAVE:
- 用 structured outputs 的 enum 把 kp_id 锁死在骨架清单的合法 ID 集合内(schema 层面禁止编造考点 ID),比“选不出来”更彻底,方案只提了 additionalProperties: false,没提 enum 约束。
- 重复检测:同一份卷传两次、同一道题出现在多份资料(讲义+错题本)时,薄弱点信号会重复计权污染学情——需按题目内容指纹去重。
- Prompt injection 防护:上传文档的文字内容会进入 prompt,恶意或偶然的指令文本(甚至印在教辅上的广告语)可能干扰输出,至少要在 prompt 中做角色隔离并对异常输出做检测。
- 公式的存储与渲染格式未定义:stem_text 里的公式以什么形式存(LaTeX/Unicode/纯文本)、确认页如何渲染,直接影响学生能否读懂确认项,应在骨架版定死约定。
- declared_intent 漏了高频场景:“做了但老师没批、自己对过答案的卷子”——既非 graded_exam 也非 wrong_book,错题判定来源(学生自报答案 vs 批改痕迹)应扩充枚举。
- 预检阶段可加姓名/准考证号区域检测并遮盖后再送 API,一并缓解合规与家长信任问题。
- 先用零提取误差的“粘贴文字”通道小规模实测一阶段映射准确率,若一阶段已达标可砍掉两阶段+自洽采样(省一半以上成本和延迟),而不是默认全套上。
- 省份卷种与教材版本差异应在骨架字段设计时就预留(如 module 按多版本映射表),而非放在+5层当补丁——字段后补会迫使全部已标注数据返工。
