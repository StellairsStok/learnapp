# 高中物理权威知识骨架设计方案

**learnapp · 知识架构设计文档 v1.0(2026-07)**
读者:负责实现的工程师 + 不写代码的创始人。工程师看数据结构和流程;创始人看每节的「为什么这样做」和「你要做什么」。

---

## 0. 一页总纲:六个核心决策

| # | 决策 | 一句话理由 |
|---|------|-----------|
| 1 | 知识边界的唯一权威 = 《普通高中物理课程标准(2017年版2025年修订)》的"内容要求"+"学业质量",不是任何教材、不是 AI 的记忆 | 考纲已废除,课标是法定命题依据;教材有 5 个版本但全部依同一课标编写 |
| 2 | 知识库物理形态 = 一批人类可读的结构化文本文件(JSON),约 200~1000 个,每个知识点一张"卡片" | 创始人自己能打开、能读、能查;机器能校验、能检索、能注入给 AI |
| 3 | 「不超纲」靠**写在卡片里的边界字段 + 每次调用时强制注入**,绝不靠模型自觉 | 模型读过全世界的物理书(含大学、竞赛内容),"自觉"每次都是重新即兴发挥,不可复查、不可修复 |
| 4 | 「不漏」靠**课标条目 ↔ 卡片的双向映射表 + 真题反向抽检**,是可机器执行的确定性检查 | 覆盖率是能算出来的数字,不是感觉 |
| 5 | 构建流程 = 三清单交叉(课标=上限、教材=粒度、真题=权重)→ Claude 批量起草 → 创始人分级人审 → 实测驱动修订 | AI 起草快但边界判断不可靠,人审必须聚焦在 AI 最容易错的字段上 |
| 6 | 骨架版锁定**人教版**做章节映射,知识树本身版本无关 | 人教版覆盖面最广;课标考点树天然版本无关,教材差异只是"目录翻译表" |

**前置任务(阻塞项)**:目前 2025 修订版课标只有非官方转载源。骨架建设可以先用 2020 修订版官方 PDF(教育科学出版社已公开)启动——两版模块结构一致——但**必须在卡片落库前拿到 2025 修订版官方 PDF 做逐条 diff**,重点核对"学业质量 5 级→3 级"的变化。这是创始人的第一个任务(见第 9 节)。

## 1. 层级结构:五层树 + 两个横切维度

### 1.1 结构总览

```
L0 板块(标签维度,5个)        力学 / 电磁学 / 热学 / 光学与振动波 / 近代物理
 └ L1 模块(6个)              必修1 / 必修2 / 必修3 / 选必1 / 选必2 / 选必3
    └ L2 单元(约30个)         可独立成章的教学单元
       └ L3 知识点(约200个)   ★ 系统的原子单位,一张知识卡片
          └ L4 细目(5类,完整版约600-800个)
             ├ 易错点
             ├ 临界条件
             ├ 常用二级结论
             ├ 物理模型
             └ 典型题型
横切维度A:21个学生必做实验(课标明文规定,单独建实验卡)
横切维度B:省份/卷种标签(挂在题目上,不挂在知识点上——考点全国统一)
```

**给创始人的解释**:把这想象成一个五层的书架编目系统。L1 模块是课标钦定的 6 本"书",L2 单元是章,L3 知识点是"一节课能讲完的最小单位"——这一层是整个产品的原子:讲课按它讲、练习按它出、错题归因归到它、学生的掌握度记在它头上。L4 细目是每个知识点下面的"弹药库"。**L3 的编号一旦定下就永不更改**,因为将来学生的所有学习记录都挂在这些编号上,改编号等于弄丢学生档案。

### 1.2 每层的真实例子

**L1 模块 → L2 单元**(以选择性必修 1 为例,对应课标三个主题):

| L2 单元 | 编号 |
|---|---|
| 动量与动量守恒定律 | `u-x1-01` |
| 机械振动 | `u-x1-02` |
| 机械波 | `u-x1-03` |
| 光的折射与全反射 | `u-x1-04` |
| 光的波动性(干涉、衍射、偏振) | `u-x1-05` |

**L2 单元 → L3 知识点**(以 `u-x1-01 动量与动量守恒定律` 为例,拆出 7 个知识点):

| L3 知识点 | 编号 |
|---|---|
| 动量、冲量与动量定理 | `kp-x1-01-001` |
| 动量守恒定律及守恒条件判断 | `kp-x1-01-002` |
| 弹性碰撞与非弹性碰撞 | `kp-x1-01-003` |
| 反冲运动与爆炸 | `kp-x1-01-004` |
| 实验:验证动量守恒定律 | `exp-x1-01` |
| 动量与能量综合问题 | `kp-x1-01-005` |
| (边缘)二维碰撞的动量矢量分析 | `kp-x1-01-006`(标记为边缘考点) |

**L3 知识点 → L4 细目**(以 `kp-x1-01-002 动量守恒定律` 为例):

| 细目类型 | 真实例子 |
|---|---|
| 易错点 | 忘记规定正方向导致符号错(本知识点第一高频错误);把"合外力为零"与"合外力做功为零"混淆,错用机械能守恒的条件判断动量守恒 |
| 临界条件 | 弹簧连接的两物体:弹性势能最大 ⟺ 两物体**共速**;子弹打木块:共速时相对位移最大,系统生热 Q = f·Δs(相对位移) |
| 二级结论 | 弹性碰撞公式 v₁′=((m₁−m₂)v₁+2m₂v₂)/(m₁+m₂);等质量弹性碰撞交换速度;完全非弹性碰撞动能损失最大 |
| 物理模型 | 人船模型(m₁s₁=m₂s₂)、子弹打木块、板块模型、弹簧连接体、反冲(火箭) |
| 典型题型 | 选择题:碰撞结果可行性三判据(动量守恒 / 动能不增 / 不穿透);计算题:碰撞+能量守恒综合(多省压轴题型);实验题:气垫导轨验证动量守恒 |

**骨架版与完整版的区别**:骨架版里,L4 细目作为**字段内嵌在 L3 卡片里**(一张卡片就是一个 JSON 文件,细目是里面的列表);完整版把高价值细目**析出为独立卡片**(有自己的 id,可以被题目、错因标签单独引用)。这样第一版工作量小,而且升级时只是"搬家",不推翻。

## 2. 知识卡片字段设计

### 2.1 字段表(JSON Schema 的人话版)

| 字段 | 类型 | 说明 | 谁最终负责 |
|---|---|---|---|
| `id` | string | 永不更改的唯一编号 | 工程师定规则 |
| `name` | string | 知识点名称 | Claude 草拟,创始人确认 |
| `module` / `unit` / `domain` | string | 所属模块/单元/板块 | 结构性字段 |
| `curriculum_refs` | array | **引用的课标"内容要求"条目原文(逐字复制)+ 内部条目编号**。⚠️ 铁律:原文只准从官方 PDF 复制粘贴,禁止 AI 凭记忆默写 | 创始人核对 |
| `statement` | string | 定义/规律的规范表述(以课标和人教版教材措辞为准) | 创始人抽审 |
| `formulas` | array | 每条公式含:LaTeX、名称、**适用条件**(公式和条件必须绑定,不允许裸公式) | 创始人抽审 |
| `prerequisites` | array | 前置知识点 id 列表(讲课排序、诊断回溯都靠它) | Claude 草拟 |
| `misconceptions` | array | 常见误区,每条含"错误认识→正确认识→诊断信号" | 创始人重点审 |
| `boundary` | object | ★★ **超纲边界:`in_scope`(考)/ `out_of_scope`(不考,含理由)/ `edge`(边缘,注明哪类省份或什么条件下出现)** | **创始人 100% 全审** |
| `models` | array | 关联物理模型 | Claude 草拟 |
| `question_types` | array | 典型题型 + 出现频率 + 常见分值位置 | 真题统计后回填 |
| `difficulty` | object | 产品内部难度 1-5(区间);课标学业质量水平(注明依据 2020 版 5 级还是 2025 版 3 级,待终核) | 创始人抽审 |
| `exam_weight` | object | 高频/中频/低频 + 依据(近 3 年 n 套卷出现 m 次) | 真题统计生成 |
| `teaching_notes` | object | 讲法要点:切入方式、先讲什么后讲什么、类比、与哪个知识点对比 | 创始人(教研)增补 |
| `textbook_map` | object | 各版本教材章节映射,骨架版只填人教版 | Claude 草拟 |
| `status` | string | `draft` → `reviewed`(创始人审过)→ `tested`(实测中经受住检验) | 流程字段 |
| `version` / `changelog` | - | 修订历史 | 自动维护 |

**为什么 `boundary` 是全卡最重要、必须 100% 人审的字段**:其余字段写错是"物理错误",AI 大概率能写对(高中物理是它训练数据的强覆盖区);但超纲边界不是物理知识,是**中国考试制度知识**——"动量守恒考不考角动量""螺旋线运动要不要定量算"——这类判断 AI 的错误率明显更高,且错的方向通常是"往深里超"。这个字段就是产品的"纲"本身。

### 2.3 示例卡片二(创始人视角:同一种卡片渲染成的阅读版)

> 创始人日常审卡看到的是下面这种自动生成的 Markdown 版,不用碰 JSON。

---

**卡片 `kp-x2-01-004` · 带电粒子在匀强磁场中的圆周运动** 〔电磁学 · 选择性必修2 · 单元:磁场〕 状态:draft

**课标依据**:〔逐字引用课标"磁场"主题中关于洛伦兹力及带电粒子在磁场中运动的条目——落库时从官方 PDF 粘贴,当前占位〕

**规律表述**:速度方向垂直于磁场的带电粒子,仅受洛伦兹力作用时做匀速圆周运动,洛伦兹力提供向心力。

**公式与适用条件**
| 公式 | 条件 |
|---|---|
| qvB = mv²/r → **r = mv/(qB)** | v ⊥ B,匀强磁场,不计重力(微观粒子默认;液滴、尘埃、小球不得默认) |
| **T = 2πm/(qB)** | 周期与速度、半径无关——本卡最重要的单一结论 |
| t = (θ/2π)·T | θ 为轨迹圆心角,弧度制或按比例算 |

**前置知识**:洛伦兹力方向(左手定则)`kp-x2-01-002`、匀速圆周运动与向心力 `kp-b2-02-003`、圆的几何(弦、中垂线、弦切角)。

**常见误区**
1. 试图计算"洛伦兹力做的功" → 洛伦兹力永远垂直速度,**不做功、不改变速率**,只改方向。诊断信号:能量方程里出现洛伦兹力项。
2. 认为速度大周期就小 → T 只由 m、q、B 决定。诊断信号:回旋加速器题答错。
3. 找不到圆心 → 圆心必在"垂直于速度的直线"与"弦的中垂线"交点。这是几何基本功,单独练。
4. 负电荷用左手定则忘记电流方向与运动方向相反。
5. 弦切角与圆心角关系用错(圆心角 = 2×弦切角)。

**超纲边界(本卡红线)**
- ✅ 考:直线边界/矩形/圆形有界磁场的几何求解;多解问题(电性未定、方向未定、周期性);速度选择器、质谱仪、回旋加速器(原理级,最大动能 Ek = q²B²R²/2m);与电场、重力的复合场综合。
- ❌ 不考:**相对论修正**(即使题面速度很大也按经典处理);**非匀强磁场中的定量计算**;讲解中不得使用微积分。
- ⚠️ 边缘:速度不垂直于 B 的**螺旋线运动**——课标不作定量要求,绝大多数省不考定量,默认只定性提一句"分解为平行和垂直分量";学生主动问才展开,并明示"考试基本不考"。

**物理模型**:单直线边界 / 平行双边界 / 圆形磁场区(沿径入射必沿径出射;r=R 时磁聚焦)/ 旋转圆(动态圆)/ 速度选择器 / 质谱仪 / 回旋加速器。

**二级结论**(使用政策:会推导前提下允许直接用)
- 同点同速率不同方向入射:轨迹圆圆心分布在以入射点为圆心、r 为半径的圆上("旋转圆"法)。
- 圆形磁场区、沿径向射入 → 沿径向射出。
- 在磁场中运动时间由圆心角决定,最长弦对应最大圆心角(同侧出射时)。

**典型题型**:几何作图求半径/时间(高频计算);多解讨论题(高区分度);装置原理题(质谱仪/回旋加速器,选择题高频);动态圆压轴(难度 5)。

**难度**:内部 2~5;课标水平:等级考范围〔2025 版 3 级制映射待终核〕。**考频**:高频(待真题统计回填)。

**讲法要点**:物理只有一句话(洛伦兹力供向心力),难点全在几何——先花一节课专练"定圆心、画轨迹、找几何关系"三步,再进题型;与电场中类平抛运动做对比表(受力恒定 vs 大小恒定方向变、做功 vs 不做功)。

---

### 2.4 示例卡片三(L4 细目卡的形态,完整版才独立成卡)

**细目卡 `d-crit-b2-02-01` · 临界条件:竖直平面圆周运动最高点(绳/内轨模型)** 〔类型:临界条件 · 挂靠:`kp-b2-02-004` 圆周运动的动力学分析〕

- **临界内容**:绳(或圆轨道内侧)在最高点:mg = mv²min/R → **v_min = √(gR)**,此时绳/轨道弹力恰为零。
- **对比模型**:轻杆(或双轨/管道)模型最高点 v_min = 0,因为杆可以支撑。**"绳杆不分"是本细目对应的第一错因。**
- **诊断信号**:学生在杆模型里也用 √(gR) 做最小速度;或在绳模型里允许 v < √(gR) 继续做圆周运动。
- **边界**:仅要求最高点/最低点及特殊点分析,不要求任意角度位置的一般解析(边缘:个别省出现过用能量法结合任意位置受力的题,归入冲刺层)。

## 3. 数据来源与构建流程

### 3.1 三清单模型

| 清单 | 来源 | 角色 | 规则 |
|---|---|---|---|
| **上限清单** | 课标"内容要求"逐条(必修+选必约 100 条,以官方 PDF 为准)+ "学业质量"分级 + 21 个学生必做实验清单 | **天花板**:任何卡片内容不得超出 | 每张卡必须引用 ≥1 条;引用不上的内容即候选超纲 |
| **基础清单** | 人教版教材目录(必修 3 册 + 选必 3 册,共 27 章)+ 每章黑体字概念/公式 | **粒度补齐**:课标写"理解匀变速直线运动规律"一句话,教材展开为 4 个公式+2 个推论,粒度靠教材 | 教材有而挂不到任何课标条目的内容 → 人工裁决(通常能挂上;真挂不上的进 `edge` 或删) |
| **权重清单** | 近 3 年真题 5~8 套(新课标 I 卷省份自命题卷为主 + 创始人目标省份的卷)逐题标注考点 | **优先级**:决定考频字段、复习排序、细目深度 | 真题里出现但前两清单没有的考法 → 最高级警报,人工裁决(要么是漏,要么是该省特色,要么是超常规压轴) |

**给创始人的解释**:课标告诉我们"最多考到哪",教材告诉我们"具体长什么样",真题告诉我们"实际爱考什么"。三张网叠在一起,漏洞才会显形:只用课标会太粗,只用教材会分不清主次,只用真题会漏掉近年没考但随时可能考的点。

## 4. 「不漏」与「不超纲」的保证机制

### 4.1 不漏:三道防线

**防线一:覆盖率矩阵(机器可执行,确定性)。** 每张卡片必须填 `curriculum_refs`。脚本每晚跑一遍:①每条课标条目 → 至少被 1 张知识点卡引用,输出未覆盖清单(目标:0);②每张卡引用的条目编号真实存在;③行为动词一致性预警(课标写"了解",卡片难度却标 4-5,提示人工复查)。**给创始人的解释**:这就像点名册——100 个学生(课标条目)每人必须有座位(卡片),点名是机器做的,一个都藏不住。但机器只能查"有没有人坐",查不出"坐的人对不对"——某条课标是否被卡片*正确地*覆盖,仍靠你在步骤 5 的人审。

**防线二:教材目录比对。** 人教版 6 册书每一节 → 必须映射到至少一张卡(允许多节对一卡)。这补的是粒度漏洞:课标一句话背后的 4 个公式不能少一个。

**防线三:真题反向抽检(最强的实证检查)。** 覆盖率 100% 只说明"纸面不漏",真题抽检才证明"实战不漏"。每季度新增 1-2 套最新真题做逐题映射,映射不上的题就是漏洞或新趋势。

**诚实声明**:不存在数学意义上"证明不漏"的方法,因为"知识点"本身没有官方总名单(考纲已取消)。上述三道防线能做到的真实水平是:**课标条目覆盖率 100%(可证明)+ 抽检真题的题目可映射率作为代理指标(目标 ≥98%,达不到就说明有漏)**。残余风险是极冷门考法和命题新趋势,靠每年更新真题清单收敛。

### 4.2 不超纲:边界为什么必须写在卡片里

**给创始人的机理解释**:Claude 读过全世界的物理教材——包括大学普物、物理竞赛、Physics Olympiad 题解。当它给一个高三学生讲动量时,"角动量守恒更优雅"这个念头就在它的知识里,随时可能说出口。而且模型每次回答都是**重新即兴发挥**:这次忍住了,下次未必;你无法给"自觉"做质检。把边界写进卡片则完全不同:

1. **每次都在场**:讲这个知识点时,系统把该卡的 `boundary` 字段原文注入提示词("禁止使用微积分;禁止提及角动量;螺旋线运动只可定性")。模型不是靠记性,是每次都被当面划重点。
2. **错误可定位、修复可全局生效**:实测发现 AI 讲了超纲内容 → 查卡 → 要么边界没写这条(补上),要么写了但模型没遵守(工程侧加拦截)。靠自觉的错误无法定位,也无法保证不复发。
3. **可做确定性的事后拦截**:生成的讲解文本过一遍关键词筛查(积分号、"求导"、"角动量"、"洛伦兹变换"等黑名单,黑名单本身也维护在卡片/全局配置里),命中即拦截重生成。这层是普通程序,100% 可靠地"抓到写过的词"。
4. **边界即产品资产**:讯飞的护城河是 4000+ 错因标签,你的护城河之一就是这套人工审定的边界标注——它不是物理知识(网上到处有),是"考试制度 × 教学法"知识(没有现成的)。

**诚实声明(不超纲的真实水平)**:即便"卡片边界注入 + 关键词拦截"双管齐下,LLM 仍会偶发地引入超纲方法或超前知识(行业经验量级是低个位数百分比,学而思学习机被投诉"给出超纲答案"就是前车之鉴;我们的架构比"裸模型"强得多,但**不承诺 100%**)。补救机制:①学生端设"这个我们没学过"一键反馈按钮,反馈直达对应卡片的修订队列;②创始人实测协议里专设"钓鱼测试"(故意问"有没有更高级的方法"),验证边界是否顶得住;③每次模型版本升级后重跑一遍固定的边界回归测试集(约 50 个诱导性问题)。

## 5. 存储形式:向创始人解释

**这套知识库物理上就是你电脑里的一个文件夹,里面约 250 个(将来 ~900 个)文本文件。** 每个文件是一张卡片。JSON 是一种"带标签的填空表"格式——花括号、引号看着像代码,其实每一行就是`"字段名": "内容"`,你双击用记事本就能打开读。为了让你读得更舒服,系统会自动把每张 JSON 卡生成一份排版好的阅读版(就是 2.3 节那种样子),你审卡审阅读版即可;要改动时,你不用手改文件,直接对 Claude Code 说"把动量守恒卡的边界里加一条 XXX",它改完你再看一眼阅读版确认。整个文件夹用 git 做版本管理——通俗说就是"每次改动自动存档,任何时候能查每张卡改过什么、能一键退回"。

**文件组织方式**:

```
learnapp/
└─ knowledge-base/
   ├─ _meta/
   │  ├─ curriculum-2025/            # 课标条目库(~100个小文件,原文逐字落库)
   │  ├─ blacklist-global.json       # 全局超纲黑名单(微积分、角动量……)
   │  ├─ coverage-matrix.csv         # 覆盖率矩阵(Excel可开,创始人直接看)
   │  └─ card.schema.json            # 卡片格式规范(机器校验用)
   ├─ b1/                            # 必修1
   │  ├─ u01-运动的描述/
   │  │  ├─ kp-b1-01-001-质点参考系.json
   │  │  └─ ...
   │  └─ u02-匀变速直线运动/ ...
   ├─ b2/  b3/  x1/  x2/  x3/        # 其余5个模块同构
   ├─ experiments/                   # 21张实验卡
   ├─ details/                       # 完整版才有:独立细目卡(按类型分子目录)
   └─ _rendered/                     # 自动生成的Markdown阅读版(创始人看这里)
```

**为什么不用数据库**:第一,你能直接打开检查是最高优先级——知识库的质检员是你;第二,几百个文件的规模根本不需要数据库;第三,文本文件配 git,每张卡的修订史天然可追溯。将来用户多了,程序会把这些文件"编译"进数据库加速查询,但**源头永远是这批你能读的文件**。

## 6. 教材版本差异的处理

**结论:骨架版锁定人教版,而且这不是妥协,是正确架构下的自然选择。**

依据(见调研摘要一):5 个版本教材全部依据同一课标编写并通过国家审定,**知识范围与深度要求一致**;差异只在章节顺序、栏目、例题情境。高考依课标命题,不依任何教材。因此:

1. **知识树和卡片内容以课标为锚,天然版本无关**——"动量守恒定律"考什么,与学生用哪版教材毫无关系。
2. 教材版本只影响一个字段:`textbook_map`(章节映射),作用是两处——讲课时说"这在你课本第几章"、按学生的学校进度排讲课顺序。
3. **骨架版只填人教版**(覆盖面最广;创始人自测阶段用哪版都行,建议直接买一套人教版纸质书当审核参照)。锁人教版的代价被严格限制在"映射表一列",不污染知识本体。
4. 完整版扩版本 = 给 `textbook_map` 加 `yueyjiao`(粤教)、`luke`(鲁科)等键,每版约 2~4 小时映射工作(Claude 起草、创始人抽查),**不需要动任何一张卡的知识内容**。唯一需要留意的真实差异:个别选学栏目、实验器材方案不同(如测电阻率的具体方案),这类差异记在实验卡的 `textbook_map` 备注里。

## 7. 骨架版 → 完整版:两档形态与升级路径

| 维度 | 骨架版(第一版必须有) | 完整版(叠加形态) |
|---|---|---|
| 卡片 | ~220 张知识点卡+实验卡,细目内嵌为字段 | 高价值细目析出为 ~700 张独立卡,可被单独引用 |

| 真题层 | 2 套卷反向抽检(验证用,不入库) | 真题索引库:每题打"知识点id+省份+年份+难度+题型"标签(注意第 5 节调研的版权边界:解析与标签是自有资产,题目本体内测自用低风险、商用需替换为自编改编题) |

## 8. 诚实声明汇总:AI 做不到什么,怎么兜底

| 环节 | AI 真实水平 | 失败模式 | 兜底机制 |
|---|---|---|---|
| 起草卡片的物理内容 | 高中物理事实与公式准确率很高,但非 100% | 偶发公式符号错、适用条件不全 | 机器校验 LaTeX 可渲染;创始人对公式字段精审;实测暴露 |
| 起草 boundary 字段 | **最弱环节**。"考不考"是制度知识,AI 会犯"以为竞赛内容也考"或"以为边缘考法不考"两个方向的错 | 边界过松→超纲;过紧→漏讲 | 该字段 100% 人审;真题抽检双向验证 |
| 引用课标原文 | **会幻觉编造条目**,禁止凭记忆引用 | 编出不存在的条目编号和措辞 | 铁律:原文只准从 PDF 复制;核对是创始人任务 |
| 运行时遵守边界 | 注入边界+关键词拦截后估计 95%+ 合规(需实测标定,无行业公开基准) | 偶发引入超纲方法、超前知识点 | 学生反馈按钮;钓鱼测试;模型升级后回归测试 |
| 批改准确性(关联系统) | 参照 Khanmigo 教训:必须"批改时注入本卡标准答案与常见误区 + 数值计算走工具",裸模型批改不可接受 | 把对判错、算术错 | 本知识骨架的卡片正是批改时注入的参照物——这是骨架的第二用途 |
| 覆盖率脚本 | 确定性,100% 可靠 | 查不出"映射得对不对" | 映射质量靠人审 + 真题抽检 |
| "各省考情"类信息 | AI 可靠性低,公开数据也不全 | 张冠李戴 | 骨架版只按"全国共性+边缘标注"处理,省份精细化推迟到有真实用户的省份再逐省人工调研 |
