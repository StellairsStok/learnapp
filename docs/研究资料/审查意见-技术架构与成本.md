OVERALL:
这是一份架构上罕见扎实的方案:模型适配层、"能用代码绝不用模型"的批改管线、标答+rubric注入、诚实清单,均符合当前最佳实践;经核实,模型与价格信息(Opus 4.8 $5/$25、Sonnet 5 优惠价至2026-08-31、Haiku 4.5 $1/$5、Batch五折、结构化输出/缓存能力)全部准确,《普通高中物理课程标准(2017年版2025年修订)》也真实存在([华南师大教材基地](https://pmc.scnu.edu.cn/a/20251127/25.html)),不是编造。但方案有两个致命盲区:一是默认 Claude API 在中国大陆随取随用——事实上大陆不在 [Anthropic 支持地区](https://www.anthropic.com/supported-countries),ToS 禁止、支付与连接皆是障碍,连"本机自测"这个第一步都需要重新论证;二是合规清单完全漏掉双减对高中学科类培训的参照适用与"拍照搜题"类功能禁令这条离产品最近的红线。最薄弱的一环不在工程而在"人":全部质量兜底(知识卡审核、超纲黑名单、rubric、每周抽检)都压在一个未经验证具备高中物理教研能力的创始人身上,且内容生产工作量被低估2-4倍。解决大陆准入、培训监管定性、教研人力三件事之后,这份架构本身值得照着做。

FACTUAL_ERRORS:
- 【最重要】方案默认 Claude API 可从中国大陆正常调用(§3.3 称'唯一的外部依赖是调用 Claude API 的网络请求',并把切国产模型定在商业化阶段)。事实:中国大陆(含港澳)不在 Anthropic 支持地区名单(https://www.anthropic.com/supported-countries),ToS 禁止在不支持地区使用且有封号先例,注册付费需境外卡,直连不稳定。'自测阶段零障碍、上线才需要换发动机'这一前提不成立。
- 考点树 exam_scope 字段设计为'合格考|等级考'并在示例中给广东学生标'等级考':'等级考/等级性考试'是 3+3 省份(沪京津浙鲁琼)术语并伴随等级赋分;广东等 3+1+2 省物理是'选择性考试'首选科目、计原始分、随高考统考。单一字段无法同时承载两种制度,直接照此建库会在非 3+3 省份产生错误的考试范围与赋分表述(方案在学生档案里写'物理原始分计入'又与之自相矛盾)。
- 版权部分'自编/改编题=版权安全区'与'引用真题标注来源并控制比例'高度存疑:改编权属于原著作权人,对真题做保留结构的深度改编不当然免责;商业性题库使用也很难落入'合理引用';国内考试机构就真题使用维权有先例。'标注来源+控制比例'不是法律上的安全港,这段会误导创始人。
- 批改/OCR 管线依赖模型自报的 transcription_confidence 阈值做门控——LLM 自报置信度不是校准概率,'低于阈值才让学生确认'这一安全阀在技术上不可靠,不能作为已解决的机制来承诺。
- '有标准答案+rubric 约束下预期 90-97% 可接受''讯飞公开数据 97%''工整手写 ~90% 可接受转写'等数字均无可核实出处、口径不明(题级还是评分点级?),属于给创始人制造精确感的锚定数字;中文手写+多步物理演算(下标、矢量符号、分数根号结构、涂改)的真实转写与逐点批改准确率初期大概率低于这些数字。
- 缓存节省'输入成本约降60-70%'的估算忽略了三件事:①提示缓存默认 TTL 仅5分钟,学生读题/演算间隔常超过5分钟,命中率会远低于'20轮里19次命中'的假设;②缓存写入加价1.25×(1小时TTL为2×);③Opus 4.8 与 Haiku 4.5 的最小可缓存前缀为4096 tokens,系统块不够长会静默不缓存。月成本数量级结论仍成立,但该节省比例与 ¥2.5-3.3/天 数字应下修重算。(注:方案中的模型命名与单价本身经与最新价目核实无误。)

MUST_FIX:
- 先解决'发动机在中国够不够得着':要么接受用海外实体/账号+网络通道开发并明确 ToS 与封号风险、密钥失效的应急预案,要么把'国产模型为主引擎、Claude 仅作教研离线辅助(出题、写标答、起草知识卡)'提前到第一天。适配层设计是对的,但'几天完成切换'低估了重调提示词+全套批改/超纲回归验证的成本,且切换时间表必须从'商业化前'改为'内测前甚至开发期'。

- 教研工作量与专业门槛重估:100-200道题含规范解答+逐点rubric+数值答案'1.5-2周'不现实(按每题30-60分钟计需100-200工时),且每道题的示意图(q-*.svg)来源完全未解决——方案自己说'一步生成SVG不可靠',那100多张图谁画?更根本:全方案的质量兜底都假设创始人具备判断超纲边界、审核教案、设计rubric的高中物理教研能力,而分工设定他是不写代码的产品经理,这一能力未验证;若不具备,需在人员计划中新增外聘/兼职物理教师角色,否则'人工审核'这道最后闸门形同虚设。
- 建立自动化评测与回归体系:提示词每周调优、未来切国产模型、模型版本升级,都需要固定的金标准测试集(批改判定准确率、超纲泄漏率、归因标签一致性)自动回归,'每周人工抽检20份'统计上撑不起质量结论;同时代码100%由 Claude Code 产出而创始人读不了代码,必须把单元测试/E2E测试+每次改动跑测试作为骨架版硬性纪律写进方案,否则每次迭代都靠人肉全流程回归。
- OCR 门控机制重设计:对计算题默认'总是回显转写结果让学生确认'(或至少对最终答案行强制确认),而不是依赖不可校准的自报置信度阈值;把'置信度'降级为排序参考而非安全机制。
