# 文档理解管线设计方案
## ——从"学生发一个文件"到"有逻辑的教学计划"

> 项目:learnapp 高考物理 AI 私教
> 模块:文档理解管线(Document Understanding Pipeline)
> 读者:实现工程师 + 创始人(教研/产品)
> 状态:设计稿,待创始人确认骨架版范围后进入开发

---

## 0. 先诚实回答创始人的问题:"能不能完美?"

**不能。任何当前技术都做不到"发一个文件就完美提取所有知识点",本方案也不会假装能做到。** 能做到的是:

- 印刷体、图片清晰的常规试卷:**绝大部分题目被正确切分和归类**(主考点识别准确率工程目标 85–92%,详见第 3 章);
- 错误**可发现、可纠正、不污染学习计划**——这是比"追求 100% 正确"更正确的设计目标;
- 随着学生确认、创始人复核的数据积累,准确率**持续爬升**。

给创始人的一个类比:一个真人名师拿到学生的试卷,也会偶尔看错一道题的考点、看不清一个潦草的字。名师的价值不在于"零失误",而在于**发现自己拿不准时会问学生一句"这道题你是这么写的吗?"**。我们把这个"拿不准就问"的机制做进系统里(第 4 章置信度分层),就能用不完美的识别,支撑可靠的教学。

### 0.1 全方案最重要的架构决策:"识别 + 映射",而不是"自由生成"

系统里已经有一份**人工审核过的权威知识骨架**(全部高考物理考点的结构化清单,约 150–200 个节点,以 2025 修订版课标"内容要求"为锚点)。因此文件处理的任务被定义为:

> **AI 只做"分拣员":看一道题,从已有的考点清单里挑出它属于哪几个格子。AI 绝不做"立法者":不允许它从文件里发明新的知识点。**

用通俗的话给创始人解释为什么这样做:

| | 方案 A:让 AI 从文件里自由总结知识点 | 方案 B(本方案):映射到已有骨架 |
|---|---|---|
| 会不会超纲 | 会。AI 可能总结出"微积分求变力做功"这种超纲表述 | **结构上不可能超纲**——清单里没有的考点,它选不出来 |
| 会不会漏 | 会,而且漏了你不知道 | 骨架本身是全集。文件只是在全集上"点亮"覆盖到的格子,没点亮的格子就是明确的"未覆盖" |
| 同一个考点会不会有八种叫法 | 会("楞次定律"/"感应电流方向判断"/"右手定则相关"各存一份,数据全碎了) | 不会。所有题目、错题、学情都挂在同一个考点 ID 上,记忆系统才能积累 |
| 错误率能不能度量 | 不能。生成任务没有标准答案 | 能。这是一个**分类问题**:准备一批标注好的题,就能算出准确率,每次改 prompt 都能回归测试 |
| 出错的形态 | 编造一个似是而非的知识点(最危险,难发现) | 把 A 考点错标成 B 考点(可发现、可纠正) |

这也是调研摘要里 Khanmigo 和 Berkeley 研究给出的同一个工程结论的变体:**让模型对照"人工编写的可信参照物"工作,而不是凭空发挥**。骨架就是我们的可信参照物。

## 1. 完整管线:六步流水线

总览(括号内为骨架版/完整版差异,详见第 6 章):

```
┌─────────────────────────────────────────────────────────────────┐
│ ① 上传与预检      文件类型判断、图片质量检查、即时打回          │
│ ② 文件解析        Claude 多模态直接读图为主(PDF 双通道互校)   │
│ ③ 题目切分        整卷 → 结构化的"题目"对象列表                │
│ ④ 考点映射(核心)逐题映射到骨架考点,输出 top-k + 置信度      │
│ ⑤ 置信度分层与汇总 高/中/低三档处置;生成覆盖与薄弱点报告      │
│ ⑥ 教学序列生成    按前置依赖排出"先补地基再讲综合"的计划      │
└─────────────────────────────────────────────────────────────────┘
```

管线各步都定义为**独立的「prompt + 输入 schema + 输出 schema」接口**,模型可整体替换(为将来切国产模型预留);所有 LLM 输出用 JSON Schema 严格校验(Claude API 的 structured outputs / `output_config.format` 保证输出必然是合法 JSON),校验失败自动重试一次,仍失败则该题标记 `parse_failed` 进入人工队列,**绝不让脏数据进库**。

### 1.1 第①步:上传与预检

**做什么:** 接收四类输入——试卷 PDF、讲义/试卷照片(JPG/PNG)、手写错题本照片、Word 文档(.docx)。以及最朴素但最可靠的第五类:**直接粘贴文字**。

**具体做法:**
1. 文件类型与大小校验(单文件 ≤ 20MB;Claude API 对 PDF 的硬限制是单请求 32MB / 600 页,我们留足余量;超页数的 PDF 拆分后分批送)。
2. **图片质量预检(便宜且重要)**:不调用大模型,用传统图像处理在毫秒级完成——
   - 分辨率检查:长边 < 1000px 直接打回"照片太小,请靠近一点重拍";
   - 模糊检测:Laplacian 方差低于阈值 → 打回"照片糊了,请对焦后重拍";
   - 过暗/过曝/大面积反光:直方图检测 → 打回并给出重拍提示("请在光线均匀处平放拍摄")。
3. PDF 判断有无文本层(可编程提取的文字):有 → 走"双通道"模式;没有(扫描版)→ 按图片处理。
4. 创建 `DocumentJob` 任务记录,后续全部异步处理(骨架版为同步等待,见第 6 章)。

**给创始人的解释:** 预检是整条管线**性价比最高的一步**。与其让 AI 硬啃一张糊照片然后错一半,不如花 0.1 秒检测出来,立刻请学生重拍。"上传后 1 秒被打回重拍" 的体验远好于 "等 30 秒得到一堆错误结果"。

**失败模式与兜底:**
- 预检阈值太严,把能用的照片打回 → 打回时提供"仍要继续处理"按钮,同时记录该样本供调阈值;
- 加密 PDF / 损坏文件 → 明确报错文案,不静默失败。

**数据结构:**

```json
{
  "job_id": "doc_20260702_0001",
  "student_id": "stu_001",
  "source_type": "pdf_text | pdf_scan | photo_print | photo_handwritten | docx | pasted_text",
  "declared_intent": "graded_exam | notes | wrong_book | unknown",
  "pages": 8,
  "status": "preprocessing | parsing | mapping | awaiting_confirm | done | failed",
  "quality_flags": ["page_3_low_light"],
  "created_at": "..."
}
```

`declared_intent` 来自上传时问学生的一个单选题("这是什么?① 老师批改过的试卷 ② 讲义/笔记 ③ 我的错题本"),一个点击就能大幅提高后续判断"哪些是错题"的准确率——**不要让 AI 猜能直接问到的事**。

### 1.2 第②步:文件解析——Claude 多模态直接读图为主

**核心取舍:为什么不用传统 OCR(如 Mathpix)做主通道?**

| | 传统 OCR → 文本 → LLM(两段式) | Claude 多模态直接读图(一段式,本方案) |
|---|---|---|
| 版面信息 | 转文本时丢失(题号缩进、图文对应、红笔批改位置全没了) | 保留。模型看到的就是学生看到的 |
| 物理示意图 | OCR 无法处理,直接丢弃 | 模型能看图并描述("斜面上物块受三个力……") |
| 手写体 | Mathpix 对潦草手写弱(调研:高难度手写样本 55% vs 多模态 84%) | 多模态模型可用上下文推理消歧,"m 和 M 哪个是物块"这类问题能靠语境解决 |
| 印刷体公式 | Mathpix 文档级公式最强(完全正确率 62.9%) | 略弱于 Mathpix,但对"识别考点"这个任务足够——考点判断不要求公式每个下标都对 |
| 工程复杂度 | 两个系统、两次失败点、要维护对齐逻辑 | 一次 API 调用 |

**结论:主通道 = Claude 多模态直接读图;Mathpix 仅作为后续"高精度公式转写"的备选插件位**(例如将来做"把这道题的解答排版成漂亮的 LaTeX"时再接)。这与调研摘要二的结论一致。

**具体做法:**
- **图片**:每页一个 base64 `image` 内容块送入 Claude(模型选型:`claude-opus-4-8`,当前 Opus 档,$5/$25 每百万 token;支持高分辨率视觉,长边最大 2576px)。上传的照片统一等比缩放到长边 ≤ 2000px 再送——高考试卷页在此分辨率下文字完全可读,同时控制图像 token 成本(全分辨率一张图最高可达约 4800 token)。
- **有文本层的 PDF(双通道互校)**:通道 A 用 pdf 库直接抽文本(近乎无损);通道 B 把 PDF 作为 `document` 块直接交给 Claude(API 原生支持 base64 PDF)。Prompt 中同时给模型"抽取的文本 + 原始页面视觉",要求以文本层为准、以视觉版面定切分和图文归属。这是印刷卷准确率最高的路径。
- **Word**:用 mammoth/python-docx 抽正文与内嵌图片,图片单独走图片通道,正文走文本通道。
- 解析与切分(下一步)在**同一次模型调用**中完成,减少一次往返。

**失败模式:**
- 双栏试卷、答题卡混排:视觉模型对双栏的阅读顺序偶尔串行 → prompt 中显式要求"本页可能为双栏,请按左栏从上到下、再右栏从上到下的顺序处理";
- 页面旋转 90°:预检阶段做方向检测并自动旋转;
- 水印/背景花纹干扰:多数情况模型能忽略,重度水印会拉低识别率,归入"必然出错"清单(第 3 章)。

### 1.3 第③步:题目切分

**做什么:** 把连续的页面内容切成一个个独立的"题目"对象,并抽取每题的结构。

**具体做法(prompt 要点):**
- 以**题号为主锚点**("1."、"12."、"三、计算题"),以版面空隙、分值标注("(10 分)")为辅助信号;
- 显式处理三种难点:**跨页题**(上一页末尾未完的题接到下一页)、**材料+多小问的组合题**(切成一个母题 + N 个子问,子问各自独立映射考点)、**选择题选项**(A/B/C/D 归属到题干,不切成独立题);
- 若 `declared_intent = graded_exam`,同时要求识别**批改痕迹**:红笔的 ✓/✗、扣分、老师批注,以及学生的手写作答内容;
- 输出严格按 schema(structured outputs 强制):

```json
{
  "questions": [
    {
      "qid": "doc_0001_q12",
      "number_label": "12",
      "question_type": "single_choice | multi_choice | fill_blank | experiment | calculation",
      "stem_text": "如图,水平导轨间距 L=0.5m,磁感应强度 B=0.4T 的匀强磁场方向竖直向下……",
      "options": {"A": "...", "B": "...", "C": "...", "D": "..."},
      "has_figure": true,
      "figure_description": "两条水平平行导轨,导体棒 ab 置于其上,磁场方向垂直纸面向里,右端接电阻 R",
      "score_value": 10,
      "student_answer_raw": "学生手写内容的转写(仅 graded_exam/wrong_book)",
      "grading_mark": "wrong | correct | partial | unknown",
      "spans_pages": [3, 4],
      "extraction_confidence": 0.93
    }
  ]
}
```

注意 `figure_description` 字段:**图不转成文字就等于丢了信息**。要求模型对每幅图输出一段面向物理教师的结构化描述(装置、方向、关键标注),这段描述是下一步考点映射时"图形题"能被正确归类的关键。

**失败模式与兜底:**
- 题号缺失或手写题号(错题本常见)→ 按视觉空隙切分,`extraction_confidence` 调低,进中/低置信度通道;
- 一题被切成两题 / 两题粘成一题 → 汇总页展示切分结果缩略图,学生可"合并/拆分"一键修正(第 4 章);
- 批改痕迹误判(把勾看成叉)→ `grading_mark` 单独设 `unknown` 值,拿不准时宁可标 unknown,让学生在确认页勾选"这题我错了没",不猜。

### 1.4 第④步:考点映射(整条管线的心脏)

**做什么:** 对每道题,从权威知识骨架中选出它考查的考点(可多个),带置信度和证据。

**知识骨架节点的数据结构(前置条件,由考点树模块提供):**

```json
{
  "kp_id": "EM-IND-03",
  "name": "法拉第电磁感应定律及感应电动势计算",
  "block": "电磁学",
  "module": "选择性必修2",
  "curriculum_ref": "2025修订版课标 3.2.2",
  "description": "理解法拉第电磁感应定律,会用 E=nΔΦ/Δt 计算感应电动势……",
  "keywords": ["感应电动势", "磁通量变化率", "E=nΔΦ/Δt", "匝数"],
  "prerequisites": ["EM-IND-01(磁通量)", "EM-FLD-02(磁场对电流的作用)", "DC-03(闭合电路欧姆定律)"],
  "difficulty_tier": 2,
  "typical_forms": ["图像题(Φ-t 转 E-t)", "线框进出磁场", "导体棒切割"]
}
```

**具体做法:两阶段映射 + 骨架清单常驻缓存。**

1. **第一阶段(粗定位)**:全部 150–200 个考点的「id + 名称 + 一句话描述 + 关键词」压成一份约 1–2 万 token 的清单,放在 system prompt 中,并打上 `cache_control: {"type": "ephemeral"}`(Claude 的 prompt caching 是前缀匹配,这份清单对所有请求字节级不变,命中缓存后读取成本约为原价的 1/10;注意 opus-4-8 的最小可缓存前缀是 4096 token,我们的清单远超此数,必然生效)。模型对每题输出候选考点 top-5。
2. **第二阶段(精判)**:把 top-5 候选的**完整节点定义**(含 description、typical_forms、prerequisites)展开注入,要求模型做最终裁决,输出:

```json
{
  "qid": "doc_0001_q12",
  "mapped_kps": [
    {
      "kp_id": "EM-IND-04",
      "role": "primary",
      "confidence": 0.92,
      "evidence": "题干核心为导体棒在导轨上切割磁感线产生电动势,E=BLv 模型"
    },
    {
      "kp_id": "DC-03",
      "role": "secondary",
      "confidence": 0.85,
      "evidence": "需用闭合电路欧姆定律求回路电流"
    },
    {
      "kp_id": "MECH-NEW-02",
      "role": "secondary",
      "confidence": 0.61,
      "evidence": "末问涉及安培力与外力的平衡分析"
    }
  ],
  "out_of_scope_flag": false,
  "notes": "若题目要求求解变速阶段的位移-时间关系,可能超出高考常规要求"
}
```

关键设计点:
- **`role: primary/secondary`**:综合题必须区分"这题主要考什么"和"顺带用到什么",否则薄弱点归因会被大量 secondary 噪声淹没;
- **`evidence` 字段强制输出**:要求模型引用题干原文短语作证据。这不是装饰——它同时提升映射质量(强迫模型对齐题面)、给学生确认页提供可读的解释、给创始人复核提供判断材料;
- **`out_of_scope_flag`**:题目明显超纲(竞赛题、大学内容)时置 true,这类题不进复习计划,单独提示学生"这题超出高考范围,可跳过";
- **置信度不能信模型自报的**。LLM 自报的 0.9 和 0.7 区分度很差。可信置信度来自 **self-consistency(自洽采样)**:同一题独立跑 3 次映射,3 次 primary 一致 → 高置信;2/3 一致 → 中置信;三次各不相同 → 低置信。骨架版为控制成本,只对第一遍自报置信度 < 0.85 的题追加采样(约 20–30% 的题),成本可控。

**成本与延迟估算(工程参考,按 opus-4-8 定价 $5/$25 每百万 token 估算,标注为估算值):** 一份 8 页整卷,解析+切分+映射全流程约消耗输入 3–6 万 token(其中骨架清单大部分走缓存读,~0.1 倍价)、输出 1–2 万 token,单份约 **¥2–8 人民币**,处理时长 1–3 分钟(异步)。对比一次线下补课 ¥200–500,成本完全可接受。完整版的批量场景可走 Batch API(非实时,五折)再省一半。

**失败模式:** 见第 3 章"多考点综合题"与"图形题"条目——这一步是全管线错误率的主要来源,兜底机制在第 4 章。

### 1.5 第⑤步:汇总输出——"这份文件覆盖了什么、暴露了什么"

**做什么:** 把逐题结果聚合成两份产物。

**产物 1:文件覆盖报告(给学生看)**

```json
{
  "job_id": "doc_20260702_0001",
  "coverage": [
    {"kp_id": "EM-IND-01", "kp_name": "磁通量", "question_count": 2, "wrong_count": 1},
    {"kp_id": "EM-IND-03", "kp_name": "法拉第定律", "question_count": 3, "wrong_count": 3}
  ],
  "blocks_touched": ["电磁学(12题)", "力学(4题)"],
  "not_covered_hint": "本卷未涉及:热学、光学、近代物理"
}
```

前端渲染成考点树上的"点亮图":覆盖的考点亮起,错题多的标红。**"没覆盖什么"和"覆盖了什么"一样重要**——这正是骨架架构的红利,自由生成方案给不出"未覆盖"清单。

**产物 2:薄弱点信号(写入学生模型,供计划引擎消费)**

每道错题产生一条薄弱点信号:`(student_id, kp_id, signal_type=wrong_in_upload, weight, source_qid, timestamp)`。注意此处只做**考点级**的薄弱标记;更细的错因归因(是概念错、公式用错还是算错)属于批改/错因模块,不在本管线范围,但 `student_answer_raw` 字段已为它保留了原料。

## 2. 每类输入的真实准确率:诚实的账本

以下数字为**基于调研引证 + 同类任务工程经验的目标区间**,不是承诺;上线后以我们自己的黄金测试集实测为准(见第 7 章)。

| 输入类型 | 文字提取 | 题目切分 | 主考点映射 | 综合评价 |
|---|---|---|---|---|
| 粘贴文字 | 100%(无提取环节) | 单题无需切分 | **88–93%** | 最可靠,骨架版主推 |
| 印刷体 PDF(有文本层) | ≈99%+ | 90–95% | 85–92% | 可靠 |
| 印刷体清晰照片 | 95–98% | 88–93% | 85–90% | 可靠 |
| 印刷体扫描件(无文本层) | 93–97% | 85–92% | 83–90% | 基本可靠 |
| **手写错题本照片** | **75–88%**(可接受转写率,依字迹) | 70–85%(题号常缺失) | 75–85% | **明显不可靠,必须走确认流程** |
| 模糊/反光/倾斜照片 | 不处理,预检打回 | — | — | — |

(手写体区间上沿引自调研摘要二:2025 年研究中多模态模型对真实学生手写数学作业的可接受转写率约 88%,高难度样本 84%;下沿对应字迹潦草者。)

### 必然出错、无法根治的情形(向创始人明说)

1. **潦草手写的符号歧义**:学生写的 "m" 与 "M"、"v" 与 "ν"、上下标粘连——人类老师也要猜。影响转写,但通常**不影响考点判断**(考点看题型结构,不看具体数值),这是本管线因祸得福之处。
2. **复杂图形题**:电路图元件多于 5–6 个、光路图多次反射折射、组合装置图——图形理解正确率约 70–85%。图看错会连带考点映射错。缓解:`figure_description` 让错误显式化(学生一眼能看出"系统把滑动变阻器看成了定值电阻")。
3. **多考点综合题的归因边界**:一道力电综合大题"主要考"什么,两位真人教师都可能意见不一。secondary 考点的召回率(该标的都标上)预计只有 60–80%。这不是模型缺陷而是任务本身的模糊性——处置办法是 primary 从严、secondary 从宽,薄弱点统计主要采信 primary。
4. **缺图即缺题**:题干为"如图所示"但照片没拍到图、或 Word 里图挂了——任何模型都无法凭空补图。检测到"题干引用图但无图"时标记 `missing_figure`,提示学生补拍。
5. **批改痕迹的暧昧**:老师只在大题总分处扣分、不逐题打叉时,系统无法判断哪个小问错了。`grading_mark=unknown` + 学生确认页兜底。
6. **非常规卷面**:双栏 + 答题卡 + 密封线装订混排的整卷扫描,切分错误率显著升高。骨架版直接不支持整卷,完整版逐步啃(见第 6 章)。

## 3. 兜底设计:置信度分层与不烦人的确认交互

### 3.1 三档处置

| 档位 | 判定条件(自洽采样后) | 处置 |
|---|---|---|
| **高置信** | 采样一致 且 confidence ≥ 0.85 且提取无质量旗标 | **自动入库**,参与计划生成;在汇总页可见、可事后修改 |
| **中置信** | 采样 2/3 一致,或 0.6 ≤ confidence < 0.85 | 进入学生**一键确认**流;确认前以"待定"状态参与计划(可用但降权) |
| **低置信** | 采样不一致,或 confidence < 0.6,或 parse_failed / missing_figure | 标记进**创始人复核队列**;该题暂不参与精确计划,只以"板块级"粗信号参与(如"电磁学可能有薄弱");不阻塞学生主流程 |

阈值(0.85/0.6)是初始拍板值,黄金测试集建好后按"高置信档实测准确率必须 ≥ 95%"反推校准。

### 3.2 学生确认交互:四条铁律

确认环节做得不好,学生做两次就再也不上传了。设计原则:

1. **一屏汇总,绝不逐题弹窗打断。** 处理完成后一个确认页,列出全部中置信项,默认全部预选"确认"。学生只需扫一眼、**对错的那几项点一下改**、按一次"完成"。目标交互成本:一份卷 < 30 秒。
2. **展示的是"人话",不是数据。** 每项形如:「第 12 题(缩略图)—— 我们认为这题考的是 **法拉第电磁感应定律**,因为题目要求"计算回路中的感应电动势"。」证据句直接复用 `evidence` 字段。修改时下拉框只给 top-5 候选 + "都不对/我不确定",不让学生面对 200 个考点的大列表。
3. **只问学生答得上来的问题。** "这题你做错了吗?"学生一定知道;"这题考的是不是楞次定律?"中上学生大多能判断——判断不了就点"不确定",流入创始人队列。**绝不问"这题的次要考点是否包含闭合电路欧姆定律"这种教研问题。**
4. **确认数据是资产,要回流。** 每次学生修正都落一条 `(qid, ai_answer, human_answer, source)` 记录。这是免费的标注数据:定期用于回归测试、prompt 迭代,完整版可用于 few-shot 示例库(把历史上映射错过的题型作为反例注入 prompt)。

### 3.3 创始人复核队列

后台一个极简列表页:题目原图 + 提取文本 + AI 的候选与证据,创始人二选一或手动指定考点,单题 10–30 秒。复核结果同样回流为标注数据。低置信题在被复核前不影响学生使用(降级为板块级信号),所以队列**允许积压**,创始人按自己的节奏清。

## 4. "有逻辑地构建教学":从考点集合到复习序列

### 4.1 原理:骨架里的前置依赖图(DAG)

骨架的每个考点节点带 `prerequisites` 字段,全体考点构成一张有向无环图(例如:磁通量 → 法拉第定律 → 导体棒模型 → 电磁感应中的力电综合)。这张图由创始人审核(第 7 章),是"有逻辑"三个字的全部来源——**逻辑不是 AI 现场想的,是教研预先编进骨架、算法照着走的**。给创始人的解释:AI 负责"这个学生哪里弱",骨架负责"物理应该按什么顺序学",两者相乘才是教学计划;任何一边缺了都会变成"看起来智能、实际乱跳"的伪个性化。

### 4.2 排序算法(确定性代码,不用 LLM)

```
输入:薄弱考点集合 W(带错题数、置信度权重)、学生已有掌握度画像 M
1. 扩栈:对 W 中每个考点,沿 prerequisites 回溯;
   前置考点若在 M 中掌握度未知或偏低 → 加入"疑似地基问题"集合 F
2. 探底:对 F 生成 2-3 道快速诊断小题(从题库按考点取),先测后教——
   地基没塌就不浪费时间重讲
3. 拓扑排序:对 W ∪ F(确认塌了的) 按 DAG 做拓扑排序,前置在前
4. 分层成课:切成"地基修复 → 单点核心 → 综合应用"三阶段,
   每个考点单元 = 微课讲解(10-15min)+ 3-5 题针对练习 + 过关判定
5. 装配约束:每日容量(如 45 分钟)、临近高考的板块优先级权重
输出:有序的教学计划(可解释:每一步都能说出"因为你第X题错了/因为它是Y的前置")
```

用确定性算法而非 LLM 排序的理由:可解释、可测试、零成本、不会抽风。LLM 只在最后为每个单元生成一句"为什么现在学这个"的人话解释。

### 4.3 完整例子:一张电磁感应试卷错了 5 道题

学生上传月考卷(graded_exam),管线识别出 5 道错题及映射:

| 错题 | 题目概要 | primary 考点 | secondary |
|---|---|---|---|
| 第3题(选择) | 线框进入磁场,判断感应电流方向 | EM-IND-02 楞次定律 | — |
| 第7题(选择) | Φ-t 图像转 E-t 图像 | EM-IND-03 法拉第定律(图像) | EM-IND-01 磁通量 |
| 第10题(填空) | E=nΔΦ/Δt 计算,ΔΦ 算错 | EM-IND-03 法拉第定律 | EM-IND-01 磁通量 |
| 第14题(计算) | 导体棒切割 + 求回路电流与安培力 | EM-IND-04 导体棒模型 | DC-03 闭合电路欧姆定律、MECH-NEW-02 受力平衡 |
| 第15题(计算) | 电磁感应过程的能量转化 | EM-IND-06 电磁感应中的能量 | MECH-EN-03 功能关系 |

**归因推理(算法自动完成):** 第7、10题的 secondary 均指向 EM-IND-01(磁通量),且第10题证据显示"ΔΦ 计算错误"——**两处独立信号指向同一个前置考点,疑似地基塌方**。沿 DAG 回溯:EM-IND-01 的前置是磁场基础(EM-FLD-01),第14题 secondary 涉及 DC-03(欧姆定律,必修3内容)——列入探底名单。

**系统生成的教学计划(呈现给学生的样子):**

> **你的电磁感应专项修复计划(预计 4 次学习,每次约 40 分钟)**
>
> **第 0 步·先摸个底(10 分钟,今天)**
> 3 道磁通量小题 + 2 道闭合电路欧姆定律小题。
> *为什么:你两道错题都栽在"磁通量变化量"上,我想确认是这个更基础的概念没吃透,还是只是粗心。测完再定下面的深度。*
>
> **第 1 课·磁通量与它的变化(若摸底 <60% 才安排)**
> 讲解 Φ=BS·cosθ 的三种变化方式(B 变、S 变、θ 变)→ 5 题梯度练习。
>
> **第 2 课·两大定律各个击破**
> 楞次定律的"阻碍"判断四步法(针对第3题)→ 法拉第定律 + 图像问题(针对第7、10题,重点:图像斜率=变化率)→ 每段配 3 题。
>
> **第 3 课·导体棒模型(电磁感应的招牌题型)**
> E=BLv → 接入回路求电流(复习欧姆定律)→ 安培力 → 受力分析。一条链讲透第14题。
>
> **第 4 课·综合与能量观点**
> 电磁感应中"外力做功 → 电能 → 内能"的账本算法(针对第15题)→ 2 道力电能综合题过关。
>
> *通过标准:每课末的过关题 ≥80% 正确,否则该考点保留在复习池,三天后换题重测。*

注意计划的每一条都可回指到"因为你错了第X题"或"因为它是Y的前置"——**可解释性是家长和学生信任一个 AI 私教的基础**,也是和"刷题 App 随机推题"的本质区别。

## 5. 骨架版 vs 完整版

### 5.1 骨架版(第一个版本就要有)

**支持的输入(刻意收窄):**
1. **粘贴文字**(单题或几题)——主推通道,零提取误差,把宝贵的调试精力全部集中在考点映射这一核心难题上;
2. **清晰照片单题**(印刷体为主,手写"能用但明确提示可能出错")——覆盖"拍一道错题发过来"这个最高频场景。

**骨架版明确不做:** 整卷 PDF、多页批量、Word、批改痕迹识别、自动错题判定(错没错由学生上传时勾选)、Batch 异步队列(单题同步处理,20–40 秒内返回)。

**骨架版必须做:** 完整的两阶段考点映射 + 置信度分层 + 学生确认页 + 复核队列 + 确认数据回流 + 黄金测试集回归脚本。**兜底机制不是完整版的奢侈品,是骨架版的地基**——因为骨架版恰恰是错误率最高、最需要学习数据的阶段。

### 5.2 完整版(逐层叠加)

| 层 | 新增能力 | 依赖 |
|---|---|---|
| +1 | 多题照片(一页讲义/半张卷子)→ 引入题目切分 | 骨架版映射已稳定(黄金集 ≥85%) |
| +2 | 整卷 PDF(文本层双通道)+ 异步任务队列 + 处理进度页 | +1 的切分已验证 |
| +3 | 批改痕迹识别(红笔✓✗、扣分)→ 自动错题判定 | 需收集带批改的真实试卷样本调试 |
| +4 | 手写错题本整本处理 + 学生手写解答转写(为错因归因供料) | 置信度体系成熟,手写确认流打磨过 |
| +5 | 覆盖报告增强(按省份卷种打标签)、Batch API 批量降本、few-shot 反例库自动装配 | 数据积累 |

**升级路径的逻辑:** 每一层只引入一个新的失败源(切分→批量→批改→手写),用上一层已验证的部分托底。反过来如果第一版就上整卷 PDF,四种失败源叠加,出了错都不知道该修哪一层。

## 7. 工程实现要点速查(给工程师)

- **模型**:`claude-opus-4-8`(视觉+推理主力,$5/$25 每百万 token,1M 上下文,高分辨率视觉 2576px);预留模型抽象层(每步管线 = prompt 模板 + 输入/输出 JSON Schema,模型 ID 配置化)以备切换国产模型。
- **结构化输出**:一律用 structured outputs(`output_config.format` 的 `json_schema`,或 SDK 的 `messages.parse()` + Pydantic),杜绝手工解析自由文本;注意 schema 需 `additionalProperties: false`。
- **Prompt caching**:骨架清单放 system prompt 尾部打 `cache_control: {"type": "ephemeral"}`;**清单必须字节级稳定**(序列化 sort keys、禁止在前缀里插时间戳/学生 ID);易变内容(题目图片、学生信息)一律放缓存断点之后。用响应的 `usage.cache_read_input_tokens` 验证命中,连续为 0 说明前缀被什么东西悄悄改了。
- **图片规格**:上传照片统一转 JPEG、长边 ≤2000px 后 base64 送 `image` 块;PDF 直接作为 `document` 块(base64,≤32MB/600页,超限拆分)。
- **多页/多题**:一页一请求、页间并行,比整卷塞一个请求更稳(单点失败可单页重试);题目 ID 全局唯一,跨页题在切分后做合并 pass。
- **自洽采样**:Claude 4.6+ 系列不接受 temperature 参数,采样多样性靠"独立请求 + prompt 中轻微改写候选顺序"实现;3 次投票取多数。
- **批量降本(完整版)**:非实时的整卷任务走 Message Batches API(50% 折扣,支持 caching),按 `custom_id` 对齐结果(结果无序返回)。
- **重试与失败语义**:schema 校验失败重试 1 次 → 仍败标 `parse_failed`;429/5xx 交给 SDK 自动退避;任何题的失败不拖垮整个 job(题级隔离)。
- **回归测试**:黄金测试集跑批脚本纳入 CI,输出 primary 准确率 / secondary 召回率 / 高置信档纯度三个指标;任何 prompt 或骨架改动必须先过回归。
- **数据表**:`document_jobs`、`extracted_questions`、`kp_mappings`(含 ai/human 双列,human 覆盖 ai)、`review_queue`、`weakness_signals`——human 修正永远不覆写 ai 原始输出,两列并存供训练分析。

## 8. 结语:把"完美"翻译成可执行的承诺

对创始人那句"能不能完美提取"的最终回答,浓缩成三句可以对外说的话:

1. **清晰的印刷体材料,系统能可靠地告诉你这份文件考了什么、你弱在哪**——个别拿不准的,它会像真老师一样"问你一句",30 秒确认;
2. **手写内容能用,但系统会诚实地标出它没把握的部分**,而不是装作全对;
3. **教学顺序永远有逻辑可循**——因为顺序不是 AI 即兴发挥,而是写死在经你(教研)审定的知识骨架里,AI 只负责判断"该走骨架上的哪条路"。

不完美,但诚实、可纠错、越用越准——这比一个吹嘘完美的黑盒,更接近一个好老师。
